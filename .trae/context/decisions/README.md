## 如何使用

### 1. 按问题类型查找

| 遇到的问题 | 查看哪个 ADR |
|-----------|-------------|
| 图表渲染问题 | ADR-001 |
| 布局方向颠倒 | ADR-002 |
| ELK vs Dagre 差异 | ADR-002 |

### 2. 按关键词查找

- **Mermaid 相关**: ADR-001
- **布局引擎**: ADR-002
- **方向映射**: ADR-002

### 3. ADR 状态说明

- ✅ **已采纳**: 当前正在使用的决策
- ⚠️ **已弃用**: 不再使用，但保留历史记录
- 🔄 **待定**: 正在讨论中的决策

## ADR 模板

创建新 ADR 时，使用以下模板：

```markdown
# ADR-XXX: [决策标题]

## 状态
[已采纳 / 已弃用 / 待定]

## 背景
[描述需要解决的问题或上下文]

## 决策
[描述最终做出的决策]

## 理由
[为什么做出这个决策]

## 替代方案
| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 方案A | ... | ... | 未采用 |
| 方案B | ... | ... | 已选择 |

## 影响
[决策对项目的影响]

## 相关文件
[关联的代码文件]

## 决策日期
YYYY-MM-DD
```

## 维护规则

1. **新增决策**: 在此索引中添加条目，创建详情文件
2. **状态变更**: 更新索引中的状态，在详情文件中说明原因
3. **定期审查**: 每月检查 ADR 状态，确保索引准确
