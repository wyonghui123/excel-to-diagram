# 用户画像

## 画像1：系统架构师

| 属性 | 描述 |
|------|------|
| 角色 | 系统架构师 |
| 核心需求 | 清晰了解系统全貌、模块间依赖关系 |
| 使用场景 | 上传架构数据Excel → 生成业务对象图/服务模块图 → 分析依赖关系 |
| 关注点 | 完整性（所有模块是否覆盖）、准确性（关系是否正确）、可读性（图表是否清晰） |
| 痛点 | Excel数据分散难以直观理解、模块间关系难以追踪 |

## 画像2：产品经理

| 属性 | 描述 |
|------|------|
| 角色 | 产品经理 |
| 核心需求 | 清晰了解"我负责的范围"与"这些范围与其他模块的关系" |
| 使用场景 | 选择中心领域 → 查看负责范围内的业务对象 → 分析跨域关系 |
| 关注点 | 范围边界（我的vs非我的）、关系清晰度、汇报友好性 |
| 痛点 | 难以快速定位自己负责的领域、跨域关系不直观、汇报需要手动整理 |

## 画像3：业务分析师

| 属性 | 描述 |
|------|------|
| 角色 | 业务分析师 |
| 核心需求 | 理解业务对象间的数据流向和依赖关系 |
| 使用场景 | 导入数据 → 按领域分组查看 → 分析数据流向 |
| 关注点 | 数据完整性、分类准确性、导出便捷性 |
| 痛点 | 数据格式不统一、分类规则不清晰 |

## PM场景核心需求

来源：docs/需求Backlog.md 第10章"产品经理场景优化"

1. **中心范围+关系范围二级选择**：PM先选中心领域，再选关系范围
2. **关系结构树形展示**：以树形结构展示"负责范围内"和"负责范围外"的关系
3. **关系分类选择**：按依赖/调用/数据/流程分类筛选关系
4. **汇报友好**：简洁模式、动态演示模式、幻灯片导出
5. **多领域快速切换**：已保存的范围配置
