## 一、产品概述

### 1.1 产品定位

**产品名称**：Excel转Diagram工具

**核心价值**：将Excel/CSV中的业务对象数据和模块关系转换为可视化架构图

**目标用户**：
- 业务分析师
- 系统架构师
- 产品经理

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| 文件解析 | 支持.xlsx、.xls、.csv格式文件，自动识别编码 |
| 数据分类 | 自动识别业务对象、服务模块、关系数据 |
| 图表渲染 | 支持Mermaid、Draw.io、Excalidraw三种引擎 |
| 颜色分组 | 支持按领域/子领域分组着色 |
| 导出分享 | 支持导出为PNG图片 |
| 备注标注 | 支持为容器、节点、关系添加编号备注说明 |

## 二、模块划分

### 2.1 模块总览

```
┌─────────────────────────────────────────────────────────────┐
│                      数据导入模块                             │

┌─────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────┘
```

### 2.2 模块详情

#### 模块1：数据导入

**定位**：接收用户上传的文件，解析为系统可用的结构化数据

**能力**：

| 能力 | 描述 | 输入 | 输出 |
|------|------|------|------|
| 文件上传 | 用户选择或拖拽上传文件 | .xlsx/.xls/.csv | File对象 |
| 文件解析 | 读取文件内容，解析Sheet数据 | File对象 | JSON数组 |
| 数据分类 | 根据Sheet名称/内容自动分类 | 原始Sheet数据 | 业务对象/服务模块/关系数据 |
| 编码检测 | 自动识别UTF-8/GBK编码 | 文件字节流 | 编码标识 |

**核心流程**：
```
用户上传文件 → 自动检测编码 → 解析Sheet数据 → 按名称/内容分类 → 输出结构化数据
```

---

#### 模块2：数据处理

**定位**：对原始数据进行验证、转换，构建适合渲染的格式

**能力**：

| 能力 | 描述 |
|------|------|
| 数据验证 | 校验必填字段、数据格式 |
| 层级构建 | 构建领域-模块-子模块的层级结构 |
| 关系映射 | 建立业务对象之间的关系 |
| 预览构建 | 生成供预览用的完整数据模型 |

**核心类型**：

```javascript
// 业务对象
{
  name: "客户",
  code: "Customer",
  domain: "客户域",
  subDomain: "客户管理"
}

// 服务模块
{
  name: "客户服务模块",
  code: "CSM",
  domain: "客户域",
  subDomain: "客户服务"
}

// 关系
{
  source: "客户",
  target: "订单",
  relationCode: "R001",
  relationDesc: "客户可创建多个订单"
}
```

---

#### 模块3：图表渲染

**定位**：将结构化数据渲染为可视化图表，提供交互能力

**图表类型**：

| 类型 | 描述 |
|------|------|
| 业务对象图 | 展示业务对象之间的关系 |
| 服务模块图 | 展示服务模块的层级结构和关系 |

**核心功能**：

| 功能 | 描述 |
|------|------|
| 节点渲染 | 根据数据生成图表节点 |
| 连线渲染 | 绘制节点间的关系连线 |
| 颜色分组 | 按领域/子领域着色区分 |
| 连线颜色规则 | 优先采用非中心节点的颜色（服务模块图） |
| 布局调整 | 容器大小、节点位置自适应 |
| 缩放拖拽 | 支持鼠标滚轮缩放、拖拽平移 |
| 颜色切换 | 切换颜色分组方式（领域/子领域） |
| 拖尾虚线 | 标签到连线的视觉连接线 |
| 标签背景 | 关系连线标签显示白色背景 |

**交互能力**：

| 交互 | 行为 |
|------|------|
| 鼠标悬停 | 显示关系详情tooltip |
| 点击节点 | 高亮关联的连线和标签 |
| 点击连线 | 高亮源节点和目标节点 |
| 点击空白 | 清除高亮状态 |
| 滚轮缩放 | 放大/缩小图表 |
| 拖拽平移 | 移动图表视图 |

---

#### 模块4：导出集成

**定位**：将图表导出为可分享的格式，支持多种输出方式

**能力**：

| 能力 | 格式 | 描述 |
|------|------|------|
| 图片导出 | PNG | 高清图片输出 |

## 三、数据模型

### 3.1 领域层级结构

```
领域 (Domain)
  └── 子领域/模块 (SubDomain/Module)
        └── 子模块 (SubModule)
              └── 业务对象 (BusinessObject)
```

### 3.2 实体关系

```
业务对象 ←1:N→ 关系 →N:1→ 业务对象
   │
   └── 属于 → 子领域/模块
```

### 3.3 服务模块关系类型

| 关系类型 | 描述 |
|----------|------|
| 依赖 | A依赖B提供的服务 |
| 调用 | A调用B的接口 |
| 数据 | A使用B的数据 |
| 流程 | A是B流程的一部分 |

## 四、用户流程

### 4.1 主流程（5步骤）

```
步骤1: 上传文件
    │
    ▼
步骤2: 选择范围（选择哪些数据参与图表）
    │
    ▼
步骤3: 选择图表类型（业务对象图/服务模块图）
    │
    ▼

```


### 5.3 BlockDiagram抽象层架构

**定位**：通用图表抽象层，为业务对象图和服务模块图提供统一的底层能力

**架构设计**：

```
useBlockDiagram/
├── model/              # 数据模型（DiagramNode, DiagramLink, DiagramContainer）
├── transform/          # 数据转换器（BusinessObjectTransformer, ServiceModuleTransformer）
├── layout/             # 布局计算（节点/容器大小预计算）
├── strategy/           # 策略模式（图表类型个性化配置）
├── syntax/             # Mermaid语法生成
├── behavior/           # 行为层（缩放、选择、tooltip）
├── style/              # 样式层（节点/连线/容器样式）
├── renderer/           # 渲染器（Mermaid渲染封装）
└── useBlockDiagram.js  # 统一入口
```

**核心能力**：

| 能力 | 描述 |
|------|------|
| 数据模型抽象 | 统一的节点、连线、容器数据结构 |
| 数据转换策略 | 支持不同图表类型的数据转换 |
| 节点大小预计算 | 根据内容自动计算节点尺寸 |
| 策略模式配置 | 每种图表类型可独立配置行为和样式 |
| 行为统一封装 | 缩放、选择、tooltip等交互行为 |
| 样式统一管理 | 节点、连线、容器样式统一处理 |

**扩展性**：
- 支持新增图表类型（只需添加新的策略配置）
- 支持自定义数据转换器
- 支持自定义行为和样式
