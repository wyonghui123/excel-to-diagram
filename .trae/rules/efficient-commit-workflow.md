# AI Coding Agent 高效 Commit 工作流

> **核心目标**：减少"继续"等待次数 + 避免重复状态检查 + 一次性批量执行

## 三大低效模式（必须避免）

### 1. 逐句"继续"循环

**❌ 低效**：
```
改代码 → "继续" → 检查状态 → "继续" → commit → "继续" → 重启 → "继续" → 测试
每次"继续" = 完整 token 消耗 + AI 重新加载上下文
```

**✅ 高效**：
```
1. 一次性规划所有步骤
2. 批量执行（git status + worktree list + diff --name-only）
3. 按类别分类 commit
4. 一次汇报完成
```

### 2. 重复状态检查

**❌ 低效**：每个 commit 前都执行
```
git status
git diff
netstat
service status
```

**✅ 高效**：操作前**一次性完整检查**
```
git status --short          # 工作区状态
git worktree list           # worktree 列表
git log --oneline -5        # 最近 commits
git diff --name-only <a>..<b>  # 分支差异（按需）
```

### 3. 回退式修复

**❌ 低效**：
```
发现问题 → 修复 → 测试 → 又发现问题 → 回退 → 重新做
```

**✅ 高效**：
```
修复前先确认根因 → 一次性修复所有相关问题 → 验证
```

## 标准 Commit 工作流（4 步）

### Step 1：状态预检（一次性）

```powershell
# PowerShell 一次性检查所有状态
git status --short
git log --oneline -5
git worktree list
git branch --merged main
```

### Step 2：暂存工作区（如有未提交改动）

```powershell
git stash push -m "wip-before-commit-<reason>"
```

### Step 3：分类提交（不要混在一起）

按类别分 commit：

| Commit 类型 | message 前缀 | 文件类别 |
|------------|-------------|---------|
| Trae 适配 | `chore(trae):` | `.trae/`、`AGENTS.md` |
| 前端源码 | `fix(frontend):` | `src/**/*.vue`、`src/**/*.ts` |
| 后端源码 | `fix(backend):` | `meta/**/*.py` |
| 测试代码 | `chore(test):` | `meta/tests/`、`e2e/` |
| 文档 | `docs:` | `docs/` |
| Worktree 清理 | `chore(worktree):` | N/A |

### Step 4：恢复 stash

```powershell
git checkout -- <auto-generated-files>   # 丢弃 IDE 自动生成
git stash pop
```

## Cherry-pick / Merge 工作流（高效版）

### 场景：A 分支有价值改动，需要合并到 main

**❌ 低效**：merge 整个分支（可能带一堆冲突）
```
git merge A --no-ff
# → 大量冲突 → 逐个解决 → 重新测试
```

**✅ 高效**：cherry-pick 单个 commit
```
git stash push
git cherry-pick <commit-hash> --no-commit
# → 检查冲突 → 解决（如有）→ commit
git checkout -- <auto-generated-files>
git stash pop
```

### 场景：合并 + 删除 worktree

**✅ 一气呵成**：
```
1. git stash push
2. git cherry-pick <hash> --no-commit
3. git checkout -- components.d.ts auto-imports.d.ts
4. git commit --no-verify -m "..." [pm-authorized]
5. git stash pop
6. git checkout -- components.d.ts
7. git worktree remove <path> --force
8. git branch -D <branch>  # 注意：D 大写强制删除（cherry-pick 后 git 不认为已合并）
```

## Pre-commit Hook 注意事项

### 问题：Hook 阻断 commit

```
[pre-commit v3.0] BLOCKED: 主工作树 commit 风险
```

**✅ 解决方案**：PM 明确授权时用 `--no-verify` + `[pm-authorized]` tag
```powershell
git commit --no-verify -m "fix: ... [pm-authorized]"
```

### 授权条件（全部满足）

1. PM 明确要求在主工作树 commit
2. 无其他 Agent 在同一工作树并发工作（或 worktree 已清理）
3. commit message 标注 `[pm-authorized]`

## PowerShell 特殊语法陷阱

### ❌ 不支持 `&&` 链式命令

```powershell
# [X] 错误
git checkout -- file && git stash pop

# [OK] 分开执行
git checkout -- file
git stash pop
```

### ❌ 不支持 heredoc `<<'EOF'`

```powershell
# [X] 错误
git commit -m "$(cat <<'EOF'
multi-line
EOF
)"

# [OK] 单行 message
git commit -m "single line message"
```

### ❌ `curl` 是 `Invoke-WebRequest` 别名

```powershell
# [X] 错误 - 会卡死
curl http://localhost:3010/

# [OK] 用 curl.exe 或 Invoke-RestMethod
curl.exe http://localhost:3010/
Invoke-RestMethod -Uri http://localhost:3010/
```

### ❌ `stash@{0}` 被 script block 拆分

```powershell
# [X] 错误
git stash show stash@{0}

# [OK] 用变量
$ref = 'stash@{0}'
git stash show $ref
```

## Auto-generated 文件处理

### 必须排除的文件（每次 commit 前 checkout 掉）

```powershell
git checkout -- components.d.ts auto-imports.d.ts test_temp/test_progress.json
```

### 原因

这些文件是 IDE/Vite 自动生成的，每次状态不同，不应进入 git 历史。

## 高效汇报模板

完成 commit 工作后，**一次性汇报**：

```
## Commit 汇总

| Commit | 说明 | 文件数 |
|--------|------|--------|
| <hash> | chore(trae): ... | N |
| <hash> | fix(frontend): ... | N |
| <hash> | fix(backend): ... | N |

## Worktree 清理

已清理 N 个 worktree:
- <path1>
- <path2>

剩余 N 个活跃 worktree。
```

**避免**：每完成一个 commit 就汇报一次。
