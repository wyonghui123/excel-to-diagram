# 文档同步规则

> 最后更新: 2026-06-07 | 状态: 活跃
> 合并自 doc-sync-rules.md 和 sync-checklist.md
> Spec 是权威来源，docs 是人类可读的聚合视图

## 同步触发条件

| 触发时机 | 同步内容 |
|---------|---------|
| Spec 创建 | 将新 spec.md 内容合并到需求文档 |
| Spec 更新 | 将变更同步到需求/设计文档的相关章节 |
| Spec 验收通过 | 更新版本记录和功能清单 |
| Checklist 完成 | 更新功能状态 |

## 文档对应关系

```
.trae/specs/{feature}       ->      docs/
|-- spec.md                  ->      需求文档.md (功能需求章节)
|-- design.md                ->      架构设计文档.md (相关模块章节)
\-- checklist.md             ->      需求文档.md (功能状态/验收标准)
```

## 同步检查清单

### Spec 创建时

- [ ] 在 docs/需求文档.md 功能清单中添加新功能条目
- [ ] 设置功能状态为「[IN_PROGRESS] 开发中」
- [ ] 记录 Spec 来源路径
- [ ] 如有新增业务规则，同步到第 8 章

### Spec 更新时

- [ ] 检查需求文档中的功能描述是否需要更新
- [ ] 如有技术设计变更，同步到 docs/架构设计文档.md
- [ ] 如有新增规则，同步到 docs/需求文档.md 第 8 章

### Spec 验收时

- [ ] 确认 checklist.md 所有项目已完成
- [ ] 将功能状态从「[IN_PROGRESS] 开发中」改为「[DONE] 已完成」
- [ ] 更新版本规划（如有必要）

### 文档审核

- [ ] 确认 docs/需求文档.md 包含该功能
- [ ] 确认 docs/架构设计文档.md 包含相关设计
- [ ] 确认链接有效（相对路径正确）
- [ ] 确认没有重复/冲突的功能定义

## 功能状态

| 状态 | 含义 |
|------|------|
| [PENDING] 待开发 | 规划中，尚未开始 |
| [IN_PROGRESS] 开发中 | 正在开发 |
| [DONE] 已完成 | 开发完成，已验收 |
| [DEPRECATED] 已废弃 | 不再维护 |
