# 工程规范规则

## 方案评估规则

### 规则 1：不确定性标记规则

**触发条件**：当方案中包含以下不确定词汇时
- "可能"
- "也许"
- "应该可以"
- "理论上"
- "推测"
- "大概"

**要求**：
1. 必须提供**完整的测试验证步骤**
2. 必须说明**如果测试失败的备选方案**
3. 必须记录**已知的不确定因素**

**示例**：
```markdown
## 方案 X

- 不确定点：wrappingWidth 可能不生效
- 验证步骤：
  1. 刷新页面
  2. 检查控制台日志
  3. 截图对比
- 失败备选：如果不生效，考虑设置 wrappingWidth: 0
```

### 规则 2：假设验证规则

**触发条件**：基于假设进行代码修改时

**要求**：
1. 修改前必须列出**所有假设**
2. 修改后必须**逐一验证**
3. 验证结果必须**记录到输出**

### 规则 3：配置生效确认规则

**触发条件**：修改配置参数后

**要求**：
1. 明确说明**配置生效的预期表现**
2. 如果配置**可能**不生效，必须提供**降级方案**
3. 添加**调试日志**确认配置是否被读取
