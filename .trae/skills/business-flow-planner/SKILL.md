# business-flow-planner

> **版本**: 1.0
> **状态**: Active
> **注册 ID**: SK-022
> **依赖**: test-gen SKILL, discover_business_rules.py

## 1. 必读上下文

- `.trae/rules/SESSION_REMINDER.md` (全局规则)
- `.trae/context/business-view.md` (业务视角)
- `meta/schemas/<object>.yaml` (业务对象 schema)
- `.trae/specs/<feat>/spec.md` (业务功能 spec)
- `.trae/specs/_business_rules/<object>.yaml` (派生的业务规则)

## 2. Pipeline

### Stage 1: 输入验证
- 确认 spec.md 存在
- 确认目标 schema 存在
- 确认 _business_rules/<object>.yaml 存在
- 如不存在,自动运行 `python .trae/scripts/discover_business_rules.py --all`

### Stage 2: 业务流生成
- 用 LLM(用户在 TRAE IDE chat 中选定的模型,如 Claude Sonnet 4)读取:
  - `spec.md` (业务功能描述)
  - `_business_rules/<object>.yaml` (业务规则清单)
- 生成 7 节 YAML:
  - `actor`: 目标用户角色
  - `goal`: 业务目标(业务可读)
  - `preconditions`: 前置条件
  - `tasks`: **业务原子动作**(不是 UI 操作)
  - `questions`: **业务断言**(不是 DOM 断言)
  - `data_tables`: 测试数据
  - `cleanup`: 清理策略

### Stage 3: IDE 集成(关键)
- 写入 `.trae/specs/<feat>/business-flow.yaml`
- 头部 banner:
  ```yaml
  # 🤖 AI Generated Draft - needs_pm_review: true
  # Review Actions: [Approve] [Request Changes] [Edit in IDE]
  # Generated by: business-flow-planner
  # Generated at: 2026-06-13T10:00:00Z
  review_status: draft
  agent_draft: true
  ```
- 调用 MCP `.trae/scripts/mcp_ide_server.py`:
  - `file_open(path)` - 在 IDE 中打开文件
  - `status_bar_set(text="📋 Business Flow Draft Ready")`
  - `show_notification(title="📋 Business Flow Draft Ready", action_buttons=[Approve, Edit, Reject])`

### Stage 4: 等待 PM Review
- 暂停在 chat 中,等用户决策:
  - **Approve** → 触发 business-flow-generator
  - **Edit in IDE** → 人工编辑 YAML
  - **Request Changes** → chat 反馈循环,LLM 重新生成

## 3. 输出

- `.trae/specs/<feat>/business-flow.yaml` (7 节结构)
- IDE 中自动打开
- 状态栏 + 通知 + 等待用户

## 4. 模板

参考 [business-flow.template.yaml](../../specs/templates/business-flow.template.yaml)

## 5. 关联文件

- 触发: `chat` 中自然语言或 `/biz-test <feat>`
- 输入: `spec.md` + `_business_rules/<object>.yaml`
- 输出: `business-flow.yaml` (draft → reviewed → approved)
- 下游: business-flow-generator
