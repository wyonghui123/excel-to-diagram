## 1. 背景与目标

### 1.1 业务背景

架构数据管理涉及多种对象的批量导入导出需求：
- **导入场景**: 批量创建/更新领域、子领域、服务模块、业务对象
- **导出场景**: 导出完整层级结构（含级联子对象）、导出选中对象、导出模板

### 1.2 组件库现状

**好消息**: 组件库中已有通用的 ImportDialog 和 ExportDialog 组件！

| 组件 | 路径 | 已有功能 |
|------|------|---------|
| **ImportDialog** | `src/components/common/ImportDialog/` | 拖拽上传、预览验证、异步导入、进度显示、模板下载 |
| **ExportDialog** | `src/components/common/ExportDialog/` | 导出范围、筛选条件、排序 |

**README 文档**: 已有完整的使用说明和 API 文档

### 1.3 能力差距分析

#### ImportDialog 需要增强的能力

| 能力 | 现有状态 | 需要增强 | 说明 |
|------|---------|---------|------|
| 多对象类型选择 | ❌ 仅支持单一 objectType | **需要** | 架构数据需要同时导入 domain、sub_domain、service_module |
| 异步导入进度条 | ❌ 无进度百分比 | **需要** | 大数据导入需要实时进度反馈 |
| 当前处理类型显示 | ❌ 无 | **需要** | 显示"正在导入子领域..." |
| 可取消导入 | ❌ 无 | **可选** | 支持取消长时间运行的导入 |
| 架构上下文参数 | ⚠️ 基础支持 | **确认** | version_id 等上下文 |

#### ExportDialog 需要增强的能力

| 能力 | 现有状态 | 需要增强 | 说明 |
|------|---------|---------|------|
| 级联导出选择 | ❌ scope=cascade 硬编码 | **需要** | 用户可选择单对象/级联导出 |
| 多对象类型选择 | ❌ 无 | **需要** | 级联导出时选择包含哪些层级 |
| 导出选项配置 | ❌ 无 | **需要** | 包含层级路径、编码/名称、保护工作表、只读标记 |
| 级联深度选择 | ❌ 无 | **可选** | 可选择导出深度 |
| 架构上下文参数 | ⚠️ 基础支持 | **确认** | version_id 等上下文 |

## 3. 功能需求

### FR-001: ImportDialog 多对象类型选择

- **描述**: ImportDialog MUST 支持同时导入多个对象类型
- **详细说明**:
  - 新增 `objectTypes` prop（数组），支持传入多个对象类型
  - UI 上显示多选复选框，让用户选择要导入的对象类型
  - 自动按依赖顺序排序（domain → sub_domain → service_module → business_object）
- **验收标准**:
  - [ ] `objectTypes` prop 支持数组类型
  - [ ] UI 显示多选复选框列表
  - [ ] 导入顺序自动按依赖关系排序
- **优先级**: Must
- **增强位置**: ImportDialog.vue
- **现有代码**: 仅支持单一 `objectType` string

### FR-002: ImportDialog 异步导入进度条

- **描述**: ImportDialog MUST 支持异步导入并显示实时进度
- **详细说明**:
  - 调用 `POST /api/v1/import/async` 启动异步导入
  - 轮询 `GET /api/v1/import/status/{task_id}` 获取进度
  - 使用 `el-progress` 显示进度条
  - 显示当前处理的对象类型
- **验收标准**:
  - [ ] 调用异步导入 API
  - [ ] 进度条实时更新（0-100%）
  - [ ] 显示当前处理对象类型
  - [ ] 完成后显示统计结果
- **优先级**: Must
- **增强位置**: ImportDialog.vue
- **现有代码**: 调用同步 `boService.importData`

### FR-003: ImportDialog 当前处理类型显示

- **描述**: ImportDialog MUST 在导入过程中显示当前处理的对象类型
- **详细说明**:
  - 从 `import/status/{task_id}` 获取 `current_type_name` 字段
  - 在进度条下方显示 "正在导入 领域..."
  - 提供导入进度的上下文信息
- **验收标准**:
  - [ ] 显示当前处理对象类型名称
  - [ ] 实时更新
- **优先级**: Must
- **增强位置**: ImportDialog.vue

### FR-004: ExportDialog 级联导出模式

- **描述**: ExportDialog MUST 支持级联导出模式选择
- **详细说明**:
  - 新增 `exportMode` prop，支持 'single' | 'cascade'
  - UI 上显示单对象/级联导出单选按钮
  - 调用 API 时传递正确的 `scope` 参数
- **验收标准**:
  - [ ] `exportMode` prop 支持选择导出模式
  - [ ] 级联导出时包含所有子对象
  - [ ] 每个对象类型独立 Sheet
- **优先级**: Must
- **增强位置**: ExportDialog.vue
- **现有代码**: scope 硬编码为 'single' 或 'cascade'

### FR-005: ExportDialog 多对象类型选择

- **描述**: ExportDialog MUST 支持级联导出时选择包含哪些对象类型
- **详细说明**:
  - 级联导出模式下，显示多选复选框
  - 复选框显示对象类型及其级联关系（如 "子领域 ← 级联自领域"）
  - 传递给 API 的 `selected_types` 参数
- **验收标准**:
  - [ ] 显示对象类型多选列表
  - [ ] 显示级联关系标签
  - [ ] 正确传递 selected_types 参数
- **优先级**: Must
- **增强位置**: ExportDialog.vue

### FR-006: ExportDialog 导出选项配置

- **描述**: ExportDialog MUST 支持丰富的导出选项
- **详细说明**:
  - 新增 `options` prop，支持以下选项：
    - `includeHierarchyPath`: 包含层级路径列
    - `includeHierarchyIds`: 包含层级编码/名称列
    - `protectSheet`: 保护工作表
    - `markReadonly`: 标记只读字段
- **验收标准**:
  - [ ] UI 显示导出选项复选框
  - [ ] 正确传递给 API 的 options 参数
- **优先级**: Should
- **增强位置**: ExportDialog.vue

### FR-007: 模板下载

- **描述**: 系统 MUST 支持下载导入模板
- **详细说明**:
  - ImportDialog 已有模板下载功能
  - 支持按选中的对象类型下载模板
- **验收标准**:
  - [ ] 点击下载生成模板
  - [ ] 模板格式正确
  - [ ] 表头与配置一致
- **优先级**: Must
- **类型映射**: 功能需求（已有）
- **来源**: 用户效率需求

### FR-008: 导入结果统计

- **描述**: ImportDialog MUST 支持详细的导入结果统计
- **详细说明**:
  - ImportDialog 已有按类型分列统计功能（created/updated/skipped/failed）
  - 进度完成后显示统计表格
- **验收标准**:
  - [ ] 显示导入结果摘要
  - [ ] 按类型分列统计
  - [ ] 错误列表可展开查看
- **优先级**: Should
- **类型映射**: 功能需求（已有）

### FR-009: 元数据驱动

- **描述**: 导入导出配置 SHOULD 从 YAML 元数据读取
- **详细说明**:
  - 对象类型映射从 hierarchies.yaml 读取
  - 导出选项从 import_export 配置读取
- **验收标准**:
  - [ ] 无硬编码对象类型名称
  - [ ] 配置变更自动生效
- **优先级**: Should
- **类型映射**: 功能需求
- **来源**: 架构原则

## 5. 外部接口需求

### IF-001: 导入 API

- **类型**: REST API
- **端点**: `POST /api/v1/import`
- **描述**: 导入数据（同步/预览）
- **请求格式**: multipart/form-data
- **请求参数**:
  ```typescript
  interface ImportRequest {
    file: File                      // Excel 文件
    mode: 'preview' | 'execute'   // 预览或执行
    conflict_strategy: 'upsert' | 'skip' | 'replace'
    version_id?: number             // 版本上下文
    product_id?: number            // 产品上下文
  }
  ```
- **响应格式**:
  ```typescript
  interface ImportPreviewResponse {
    success: boolean
    data: {
      sheets: Array<{
        name: string
        row_count: number
        has_errors: boolean
      }>
      validation: {
        errors: Array<{
          row: number
          sheet: string
          field: string
          value: string
          message: string
        }>
      }
    }
  }

  interface ImportExecuteResponse {
    success: boolean
    data: {
      results: {
        [object_type: string]: {
          created: number
          updated: number
          skipped: number
          failed: number
        }
      }
      errors: Array<{
        row: number
        message: string
      }>
    }
  }
  ```
- **来源**: 后端 API 设计

### IF-002: 异步导入 API

- **类型**: REST API
- **端点**: `POST /api/v1/import/async`
- **描述**: 启动异步导入任务
- **请求格式**: multipart/form-data
- **响应**: `{ success: true, data: { task_id: string } }`

### IF-003: 导入状态查询 API

- **类型**: REST API
- **端点**: `GET /api/v1/import/status/{task_id}`
- **响应格式**:
  ```typescript
  interface ImportStatusResponse {
    success: boolean
    data: {
      status: 'pending' | 'processing' | 'completed' | 'failed'
      progress: number              // 0-100
      current_type: string          // 当前处理的对象类型
      total_types: number           // 总对象类型数
      current_index: number         // 当前索引
      message: string
      result?: ImportExecuteResponse
      error?: string
    }
  }
  ```
- **来源**: 后端 API 设计

### IF-004: 导出 API

- **类型**: REST API
- **端点**: `POST /api/v1/export`
- **描述**: 导出数据
- **请求格式**: application/json
- **请求参数**:
  ```typescript
  interface ExportRequest {
    object_type: string            // 起始对象类型
    scope: 'single' | 'cascade' | 'selected'
    selected_types?: string[]       // 当 scope=selected 时
    filters?: Record<string, any>  // 筛选条件
    sort_by?: string
    sort_order?: 'asc' | 'desc'
    options?: {
      include_hierarchy_path: boolean
      include_hierarchy_ids: boolean
      include_metadata_sheet: boolean
      protect_sheet: boolean
    }
  }
  ```
- **响应格式**:
  ```typescript
  interface ExportResponse {
    success: boolean
    data: {
      file_path: string
      download_url: string
      sheets: Array<{
        name: string
        row_count: number
      }>
      total_rows: number
    }
  }
  ```
- **来源**: 后端 API 设计

##### 增强 2: 异步导入进度条

**修改 startImport 方法**:
```javascript
async function startImport() {
  currentStep.value = 2
  importing.value = true
  importProgress.value = 0
  importResult.value = null

  try {
    // 启动异步导入
    const { task_id } = await boService.importDataAsync(
      props.objectType,
      selectedFile.value,
      {
        conflictStrategy: conflictStrategy.value,
        object_types: selectedTypes.value,
        version_id: props.context?.version_id
      }
    )

    // 轮询进度
    await pollImportProgress(task_id)
  } catch (e) {
    ElMessage.error('导入失败: ' + e.message)
    importResult.value = { success: false, errors: [{ message: e.message }] }
  } finally {
    importing.value = false
  }
}

async function pollImportProgress(taskId) {
  while (true) {
    const status = await boService.getImportStatus(taskId)

    if (status.success) {
      const data = status.data
      importProgress.value = data.progress || 0
      currentImportType.value = data.current_type_name || ''

      if (data.status === 'completed') {
        importResult.value = {
          success: true,
          results: data.result?.results || {},
          errors: data.result?.errors || []
        }
        ElMessage.success(`成功导入 ${successCount.value} 条数据`)
        emit('success', importResult.value)
        break
      } else if (data.status === 'failed') {
        importResult.value = {
          success: false,
          errors: [{ message: data.error || '导入失败' }]
        }
        ElMessage.error(data.error || '导入失败')
        break
      }
    }

    await new Promise(resolve => setTimeout(resolve, 1000))
  }
}
```

const cascadeMapping = {
  domain: null,
  sub_domain: '领域',
  service_module: '子领域',
  business_object: '服务模块'
}

const cascadeObjectTypes = computed(() => {
  return props.objectTypes.map(type => ({
    type,
    label: objectTypeLabels[type] || type,
    cascadeFrom: cascadeMapping[type],
    selected: true
  }))
})

const selectedCascadeTypes = computed(() => {
  return cascadeObjectTypes.value.filter(o => o.selected).map(o => o.type)
})

async function handleExport() {
  exporting.value = true

  try {
    const params = {
      object_type: props.objectType,
      scope: localExportMode.value === 'cascade' ? 'cascade' : 'single',
      selected_types: localExportMode.value === 'cascade' ? selectedCascadeTypes.value : undefined,
      filters: withFilters.value && hasFilters.value ? props.filters : {},
      options: localOptions.value
    }

    const result = await boService.exportData(props.objectType, params)

    if (result.success) {
      ElMessage.success(`导出成功，共 ${result.total_rows} 条数据`)
      emit('success', { count: result.total_rows })
      handleClose()
    } else {
      ElMessage.error(result.message || '导出失败')
    }
  } catch (e) {
    ElMessage.error('导出失败: ' + e.message)
  } finally {
    exporting.value = false
  }
}
