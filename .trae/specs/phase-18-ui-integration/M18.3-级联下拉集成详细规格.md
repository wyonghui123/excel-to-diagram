## 1. 背景与目标

### 1.1 业务背景

当前架构数据管理平台在创建业务对象（Business Object）时，用户需要手动选择所属的层级归属关系：
- 产品 (Product)
- 版本 (Version)
- 领域 (Domain)
- 子领域 (SubDomain)
- 服务模块 (ServiceModule)

这种手动选择存在以下问题：
1. **用户体验差**: 用户需要记住完整的层级路径
2. **数据一致性风险**: 手动选择可能导致层级关系不匹配
3. **效率低下**: 每次创建都需要从顶级开始选择

### 1.2 技术背景

`useCascadeSelect.js` Composable 已开发完成，提供通用的级联下拉逻辑：
- 支持父字段变更时自动清空下游字段
- 支持级联加载子选项
- 支持按需加载（懒加载）

**已完成部分**:
- `src/composables/useCascadeSelect.js` - 级联逻辑 Composable
- M18.1 已在 domain/sub_domain/service_module 配置 cascade_select

**待完成部分**:
- business_object.yaml 补充 cascade_select 配置
- MetaForm 级联下拉渲染
- 字段分组与隐藏逻辑
- 编辑时反向推断机制

### 1.3 目标声明

| 目标类型 | 目标描述 | 成功标准 |
|---------|---------|---------|
| **架构迁移** | 将级联下拉能力集成到 MetaForm | 支持 5 级级联：产品→版本→领域→子领域→服务模块 |
| **通用能力** | 沉淀 useCascadeSelect 为通用级联逻辑 | 组件可独立于业务对象场景使用 |
| **用户体验** | 简化层级归属选择流程 | 用户从树节点创建时，中间级字段自动隐藏 |

## 3. 功能需求

### FR-001: 级联下拉配置声明

- **描述**: 系统 MUST 支持在 YAML 配置文件中声明级联下拉关系
- **详细说明**:
  - 每个级联字段声明其父字段和父对象类型
  - 级联链自动从配置中构建
  - 支持配置字段显示名称、占位符等 UI 属性
- **验收标准**:
  - [ ] business_object.yaml 包含完整的 cascade_select 配置
  - [ ] 级联链覆盖 5 级：domain_id → sub_domain_id → service_module_id → product_id → version_id
  - [ ] 每个配置项包含 field、parentObject、displayField、parentField 字段
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 架构需求文档

### FR-002: 级联下拉渲染

- **描述**: MetaForm MUST 支持根据 cascade_select 配置渲染级联下拉组件
- **详细说明**:
  - 自动识别需要级联渲染的字段
  - 注入级联选项加载逻辑
  - 显示加载状态
  - 支持禁用状态（父字段未选择时）
- **验收标准**:
  - [ ] 级联字段渲染为下拉选择器（非普通输入框）
  - [ ] 父字段变更时自动触发子字段选项重新加载
  - [ ] 显示加载动画（loading spinner）
  - [ ] 父字段未选择时子字段禁用
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: FR-001 的实现需求

### FR-003: 下游字段自动清空

- **描述**: 当父字段值变更时，系统 MUST 自动清空所有下游级联字段的值
- **详细说明**:
  - 父字段变更时，递归清空所有子字段
  - 清空操作触发表单值变更事件
  - 支持配置是否清空下游（clearDownstream 参数）
- **验收标准**:
  - [ ] domain_id 变更时清空 sub_domain_id、service_module_id
  - [ ] sub_domain_id 变更时清空 service_module_id
  - [ ] 清空后触发表单验证更新
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 级联下拉标准行为

### FR-004: 字段分组渲染

- **描述**: 级联下拉字段 MUST 自动归入"层级归属"分组区块
- **详细说明**:
  - 通过 fieldGroup 配置标识分组
  - 分组区块包含标题和字段列表
  - 支持分组内字段排序（fieldGroupPosition）
- **验收标准**:
  - [ ] "层级归属"区块显示在表单中
  - [ ] 包含 domain_id、sub_domain_id、service_module_id 三个字段
  - [ ] 字段按 fieldGroupPosition 排序
- **优先级**: Should
- **类型映射**: 功能需求
- **来源**: 用户体验需求

### FR-005: 中间级字段自动隐藏

- **描述**: 当用户从树节点创建业务对象时，系统 SHOULD 自动隐藏已确定的层级字段
- **详细说明**:
  - 从树节点获取当前上下文（parent_type, parent_id）
  - 根据上下文推断需要填充的字段
  - 隐藏已填充的中间级字段
  - 仅显示需要用户选择的字段
- **验收标准**:
  - [ ] 从 SubDomain 节点创建时：隐藏 domain_id、sub_domain_id，仅显示 service_module_id
  - [ ] 从 Domain 节点创建时：隐藏 domain_id，仅显示 sub_domain_id、service_module_id
  - [ ] 从 Version 上下文创建时：隐藏 version_id，仅显示 domain_id、sub_domain_id、service_module_id
- **优先级**: Should
- **类型映射**: 用户需求
- **来源**: 用户体验优化需求

### FR-006: 编辑时反向推断

- **描述**: 编辑已有业务对象时，系统 MUST 支持从当前字段值反向推断完整层级路径
- **详细说明**:
  - 加载当前对象的服务模块 ID
  - 查询服务模块获取子领域 ID
  - 递归查询获取完整层级路径
  - 填充级联下拉的每个选项
- **验收标准**:
  - [ ] 编辑模式下自动加载完整层级路径
  - [ ] 所有级联字段的选项正确填充
  - [ ] 已选择的值正确回显
  - [ ] 上游字段不可编辑（只读）
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 完整 CRUD 功能需求

### FR-007: 级联加载防抖

- **描述**: 级联选项加载 SHOULD 支持防抖机制，避免频繁请求
- **详细说明**:
  - 用户输入时等待 300ms 再发起请求
  - 取消未完成的请求
  - 显示加载状态
- **验收标准**:
  - [ ] 快速切换父字段时仅发起一次子选项请求
  - [ ] 请求取消时不清空已加载的选项
- **优先级**: Could
- **类型映射**: 非功能需求
- **来源**: 性能优化需求

## 4. 非功能需求

### NFR-001: 性能要求

- **描述**: 级联选项加载响应时间应在合理范围内
- **测量方法**:
  - 单次选项加载（无缓存）：< 500ms
  - 带缓存的选项加载：< 50ms
  - 5 级级联完整加载：< 1s
- **优先级**: Should
- **来源**: 用户体验要求

### NFR-002: 可用性要求

- **描述**: 级联下拉交互应符合用户预期
- **测量方法**:
  - 父字段变更后 500ms 内显示子选项或加载状态
  - 禁用状态的子字段视觉上清晰可见
  - 加载状态明确指示给用户
- **优先级**: Must
- **来源**: 用户体验要求

### NFR-003: 可访问性要求

- **描述**: 级联下拉组件应支持键盘操作
- **测量方法**:
  - Tab 键可聚焦到级联下拉
  - 上下箭头可选择选项
  - Enter 键确认选择
- **优先级**: Should
- **来源**: 无障碍访问要求

### NFR-004: 可测试性要求

- **描述**: 级联下拉逻辑应易于单元测试
- **测量方法**:
  - useCascadeSelect Composable 有 90% 以上的测试覆盖率
  - 核心逻辑（选项构建、链式清空）可独立测试
- **优先级**: Should
- **来源**: 代码质量要求

- **类型**: REST API
- **端点**: `GET /api/v2/bo/{object_type}/options`
- **描述**: 查询指定对象类型的下拉选项

  ```typescript
  interface CascadeOptionsRequest {
    object_type: string      // 对象类型，如 'sub_domain'

  }

  ```typescript
  interface CascadeOptionsResponse {

  }

## 7. 约束与假设

### 7.1 技术约束

- **约束 1**: 必须使用 Vue 3 Composition API
- **约束 2**: 级联逻辑必须通过 useCascadeSelect Composable 封装
- **约束 3**: 禁止在组件中硬编码级联关系

- **约束 1**: 级联层级深度不超过 5 级
- **约束 2**: 每个对象最多只有一个父级级联关系
- **约束 3**: 版本上下文在级联链中作为第一级

### 7.3 假设

- **假设 1**: 后端 API 已支持级联选项查询（待确认） - 来源: 假设
- **假设 2**: 现有的 MetaForm 组件可以扩展支持级联渲染 - 来源: 代码分析
- **假设 3**: 级联配置存储在 YAML 文件中（不存储在数据库） - 来源: 架构设计

### 9.3 详细设计

#### 9.3.1 数据模型

```typescript
// 级联配置类型
interface CascadeConfig {
  field: string                    // 当前字段名
  parentObject: string            // 父对象类型
  displayField: string             // 显示字段名
  parentField: string              // 父字段名
  filter?: Record<string, any>     // 额外过滤条件
  clearDownstream?: boolean       // 是否清空下游
  disabled?: boolean              // 是否禁用
}

// 级联链结构
interface CascadeChain {
  [fieldName: string]: CascadeConfig
}

// 级联状态
interface CascadeState {
  options: Record<string, Array<{ id: number; name: string }>>
  loading: Record<string, boolean>
  error: Record<string, string | null>
}
```

#### 9.3.2 useCascadeSelect API 设计

> **注意**: 现有 `src/composables/useCascadeSelect.js` 已实现大部分功能。以下是现有实现的摘要和增强建议。

**现有实现摘要** (src/composables/useCascadeSelect.js):

```javascript
// src/composables/useCascadeSelect.js

import { computed, ref, watch } from 'vue'
import { boService } from '@/services/boService'

/**
 * [COMPOSABLE] 级联下拉组合函数
 * [STATUS] 现有实现已完成 80%，仅需增强反向推断功能
 */
export function useCascadeSelect(metaObject) {
  
  // ============================================================
  // [SECTION] 响应式状态
  // ============================================================
  
  const options = ref({})     // 选项缓存
  const loading = ref({})     // 加载状态
  const error = ref({})       // 错误状态
  
  // 从 metaObject.value.cascade_select 构建级联链
  const cascadeConfig = computed(() => {
    return metaObject.value?.cascade_select || []
  })
  
  const cascadeChain = computed(() => {
    const chain = {}
    cascadeConfig.value.forEach(config => {
      chain[config.field] = {
        field: config.field,
        parentField: config.filter_by,           // 使用 filter_by 作为父字段
        parentObject: config.parent_object,
        displayField: config.parent_display_field || 'name'
      }
    })
    return chain
  })
  
  const cascadeFields = computed(() => Object.keys(cascadeChain.value))
  const parentFields = computed(() => {
    return cascadeFields.value
      .map(fieldId => cascadeChain.value[fieldId]?.parentField)
      .filter(Boolean)
  })
  
  // ============================================================
  // [SECTION] 核心方法（现有实现）
  // ============================================================
  
  /**
   * [FUNCTION] loadCascadeOptions
   * [EXISTING] 加载指定字段的级联选项
   */
  async function loadCascadeOptions(fieldId, parentValue) {
    const config = cascadeChain.value[fieldId]
    if (!config || !parentValue) {
      options.value[fieldId] = []
      return []
    }
    
    loading.value[fieldId] = true
    error.value[fieldId] = null
    
    try {
      const params = {}
      params[config.parentField] = parentValue
      params.pageSize = 1000
      const result = await boService.query(config.parentObject, { filters: params })
      
      if (result.success) {
        const items = result.data?.items || result.data || []
        options.value[fieldId] = items.map(item => ({
          value: item.id,
          label: item[config.displayField] || item.name || item.code || item.id,
          _raw: item
        }))
      } else {
        error.value[fieldId] = result.message || '加载选项失败'
        options.value[fieldId] = []
      }
    } catch (e) {
      error.value[fieldId] = e.message || '网络错误'
      options.value[fieldId] = []
    } finally {
      loading.value[fieldId] = false
    }
    
    return options.value[fieldId]
  }
  
  /**
   * [FUNCTION] loadAllCascadeOptions
   * [EXISTING] 加载所有级联选项
   */
  async function loadAllCascadeOptions(formData) {
    const promises = cascadeFields.value.map(async fieldId => {
      const config = cascadeChain.value[fieldId]
      const parentValue = formData[config.parentField]
      if (parentValue) {
        await loadCascadeOptions(fieldId, parentValue)
      }
    })
    await Promise.all(promises)
  }
  
  /**
   * [FUNCTION] clearDownstream
   * [EXISTING] 清空下游字段选项
   */
  function clearDownstream(fieldId) {
    const fieldIds = cascadeFields.value
    const startIndex = fieldIds.indexOf(fieldId)
    if (startIndex === -1) return
    
    fieldIds.slice(startIndex + 1).forEach(downstreamFieldId => {
      const downstreamConfig = cascadeChain.value[downstreamFieldId]
      if (downstreamConfig && downstreamConfig.parentField === config.field) {
        options.value[downstreamFieldId] = []
      }
    })
  }
  
  /**
   * [FUNCTION] watchParentChanges
   * [EXISTING] 监听父字段变化
   */
  function watchParentChanges(formData, callback) {
    return watch(
      () => parentFields.value.map(f => formData[f]),
      (newValues, oldValues) => {
        parentFields.value.forEach((parentField, index) => {
          const newValue = newValues[index]
          const oldValue = oldValues ? oldValues[index] : undefined
          
          if (newValue !== oldValue) {
            cascadeFields.value.forEach(fieldId => {
              const config = cascadeChain.value[fieldId]
              if (config && config.parentField === parentField) {
                callback(fieldId, newValue)
              }
            })
          }
        })
      },
      { deep: true }
    )
  }
  
  // ============================================================
  // [SECTION] 需要增强的方法
  // ============================================================
  
  /**
   * [ENHANCE] inferParentFields - 反向推断父字段值
   * [DONE] v1.2.0 已实现
   * 
   * @param {string} fieldName - 字段名
   * @param {any} currentValue - 当前字段值
   * @returns {Promise<Object>} 推断结果 { success, data }
   */
  async function inferParentFields(fieldName, currentValue) {
    const config = cascadeChain.value[fieldName]
    if (!config) return { success: false }
    
    try {
      // 查询当前对象获取父字段值
      const result = await boService.read(config.parentObject, currentValue)
      
      if (!result.success || !result.data) {
        return { success: false, message: '未找到数据' }
      }
      
      const parentValue = result.data[config.parentField]
      
      // 递归推断上级字段
      if (parentValue && config.parentField) {
        const parentResult = await inferParentFields(config.parentField, parentValue)
        
        return {
          success: true,
          data: {
            ...parentResult.data,
            [config.parentField]: parentValue,
            [fieldName]: currentValue
          }
        }
      } else {
        return {
          success: true,
          data: { [fieldName]: currentValue }
        }
      }
    } catch (error) {
      return { success: false, message: error.message }
    }
  }
  
  /**
   * [ENHANCE] getDownstreamFields - 获取下游字段列表
   * [DONE] v1.2.0 已实现
   * 
   * @param {string} fieldName - 字段名
   * @returns {string[]} 下游字段列表
   */
  function getDownstreamFields(fieldName) {
    const downstream = []
    const chain = cascadeChain.value
    
    function collect(field) {
      Object.entries(chain).forEach(([key, config]) => {
        if (config.parentField === field && !downstream.includes(key)) {
          downstream.push(key)
          collect(key)
        }
      })
    }
    
    collect(fieldName)
    return downstream
  }
  
  // ============================================================
  // [SECTION] 导出接口
  // ============================================================
  
  return {
    // 状态
    cascadeConfig,
    cascadeChain,
    cascadeFields,
    cascadeDepth: computed(() => cascadeFields.value.length),
    isEmptyChain: computed(() => cascadeFields.value.length === 0),
    
    // 查询方法
    getOptions: (fieldId) => options.value[fieldId] || [],
    isLoading: (fieldId) => loading.value[fieldId] || false,
    getError: (fieldId) => error.value[fieldId] || null,
    isCascadeField: (fieldId) => cascadeFields.value.includes(fieldId),
    getParentField: (fieldId) => cascadeChain.value[fieldId]?.parentField || null,
    
    // 操作方法
    loadCascadeOptions,
    loadAllCascadeOptions,
    clearDownstream,
    clearAllDownstream: (fieldId) => {
      const fieldIds = cascadeFields.value
      const startIndex = fieldIds.indexOf(fieldId)
      if (startIndex === -1) return
      fieldIds.slice(startIndex).forEach(fid => {
        options.value[fid] = []
      })
    },
    
    // 增强方法
    inferParentFields,
    getDownstreamFields,
    
    // 监听器
    watchParentChanges,
    
    // 原始状态（用于调试）
    options,
    loading,
    error
  }
}

/**
 * [COMPOSABLE] 表单级联组合函数
 * [EXISTING] 封装 useCascadeSelect 并自动监听表单变化
 */
export function useFormCascade(metaObject, formData) {
  const cascade = useCascadeSelect(metaObject)
  
  cascade.watchParentChanges(formData, (fieldId, newValue) => {
    if (newValue) {
      cascade.loadCascadeOptions(fieldId, newValue)
    } else {
      cascade.options.value[fieldId] = []
    }
    cascade.clearDownstream(fieldId)
  })
  
  async function initialize() {
    await cascade.loadAllCascadeOptions(formData.value)
  }
  
  return {
    ...cascade,
    initialize
  }
}

  const groups = {}
  const fields = props.metaObject?.fields || []
  
  fields.forEach(field => {

function isHiddenField(field) {
  // 1. 显式配置隐藏
  if (field.ui?.visible === false) return true
  
  // 2. 从父上下文推断隐藏
  if (props.parentContext) {
    const hiddenFields = getInferredHiddenFields()
    if (hiddenFields.includes(field.id)) return true
  }
  
  return false
}

function getInferredHiddenFields() {
  if (!props.parentContext) return []
  
  const { parentType, parentId } = props.parentContext
  const hiddenFields = []
  
  // 根据父上下文类型推断隐藏字段
  switch (parentType) {
    case 'sub_domain':
      hiddenFields.push('domain_id', 'sub_domain_id')
      break
    case 'domain':
      hiddenFields.push('domain_id')
      break
    case 'version':
      hiddenFields.push('version_id')
      break
    case 'product':
      hiddenFields.push('product_id')
      break
  }
  
  return hiddenFields
}

// ============================================================
// [SECTION] 级联字段禁用判断
// ============================================================

function isCascadeDisabled(field) {
  if (!cascade.isCascadeField(field.id)) return false
  
  const parentField = cascade.getParentField(field.id)
  
  // 编辑模式下，上游字段禁用
  if (props.mode === 'edit') {
    const chain = cascade.cascadeChain.value
    let currentField = field.id
    
    // 检查是否为上游字段
    while (chain[currentField]?.parentField) {
      if (chain[currentField].parentField === parentField) {
        return true // 上游字段，禁用
      }
      currentField = chain[currentField].parentField
    }
  }
  
  // 创建模式下，父字段未选择时禁用
  if (!parentField) return false
  
  return !formData.value[parentField]
}

// ============================================================
// [SECTION] 级联事件处理
// ============================================================

async function onCascadeChange(fieldId, newValue) {
  // 1. 清空下游字段
  cascade.clearDownstream(fieldId, (downstreamField) => {
    formData.value[downstreamField] = null
  })
  
  // 2. 重新加载子字段选项
  if (newValue) {
    await cascade.loadCascadeOptions(fieldId, newValue)
  }
  
  // 3. 触发变更事件
  emit('change', { field: fieldId, value: newValue, formData: formData.value })
}

// ============================================================
// [SECTION] 编辑模式初始化
// ============================================================

async function initEditMode() {
  if (props.mode !== 'edit') return
  
  // 找到最末级级联字段作为起始点
  const chain = cascade.cascadeChain.value
  let leafField = null
  let maxDepth = 0
  
  Object.entries(chain).forEach(([field, config]) => {
    let depth = 0
    let current = config
    
    while (current?.parentField && chain[current.parentField]) {
      depth++
      current = chain[current.parentField]
    }
    
    if (depth > maxDepth && formData.value[field]) {
      maxDepth = depth
      leafField = field
    }
  })
  
  // 反向推断完整层级路径
  if (leafField && formData.value[leafField]) {
    const result = await cascade.inferParentFields(leafField, formData.value[leafField])
    
    if (result.success) {
      // 填充推断结果
      Object.entries(result.data).forEach(([field, value]) => {
        formData.value[field] = value
      })
      
      // 加载所有选项
      await cascade.loadAllOptions(formData.value)
    }
  }
}

// ============================================================
// [SECTION] 生命周期
// ============================================================

onMounted(() => {
  initEditMode()
})

// 监听父上下文变化，重新初始化
watch(() => props.parentContext, () => {
  if (props.parentContext) {
    // 从父上下文自动填充
    const hiddenFields = getInferredHiddenFields()
    
    hiddenFields.forEach(fieldId => {
      if (props.parentContext.parentId) {
        // 尝试从上下文获取值
        const value = getValueFromContext(fieldId, props.parentContext)
        if (value) {
          formData.value[fieldId] = value
        }
      }
    })
    
    // 加载可见字段的选项
    cascade.loadAllOptions(formData.value)
  }
})

    'textarea': 'el-input',

    }
  }
  
  return {}
}

| `clear_downstream` | 变更时是否清空下游 | 通常为 `true`（除第一级外） |

### 9.4 替代方案考虑

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| **方案 A**: 全部在 MetaForm 内实现 | 代码集中 | 难以复用 | **已选** |
| 方案 B: 分离级联选择器组件 | 高度复用 | 增加组件数量 | 备选 |
| 方案 C: 后端实现级联逻辑 | 简化前端 | 增加 API 复杂度 | 备选 |

**决策理由**: 方案 A 平衡了复用性和实现复杂度，通过 useCascadeSelect Composable 封装复用逻辑，同时保持 MetaForm 的简洁性。

### 10.2 版本上下文集成方式

**结论**: 通过 YAML `context` 配置和 `hierarchies.yaml` 定义实现

**层级结构** (hierarchies.yaml):
```
层级 0: product (产品)
层级 1: version (版本) ← parent_object: product, foreign_key: product_id
层级 2: domain (领域) ← parent_object: version, foreign_key: version_id
层级 3: sub_domain (子领域) ← parent_object: domain, foreign_key: domain_id
层级 4: service_module (服务模块) ← parent_object: sub_domain, foreign_key: sub_domain_id
层级 5: business_object (业务对象) ← parent_object: service_module, foreign_key: service_module_id
```

**版本上下文配置** (business_object.yaml 第 31-33 行):
```yaml
context:
  field: version_id
  description: "版本上下文，所有查询自动注入 version_id 过滤"
```

**级联依赖配置** (business_object.yaml 字段定义中):

ui:
  depends_on: null  # 版本是第一级
  cascade_level: 1

# domain_id 字段
ui:
  depends_on: version_id  # 依赖版本
  cascade_level: 2

### 10.3 缓存策略已确定

**结论**: 使用内存缓存，页面切换时自动失效

**当前实现** (useCascadeSelect.js):
```javascript
// 使用 Vue ref 存储选项（内存缓存）
const options = ref({})  // 第 5 行
const loading = ref({})  // 第 6 行
const error = ref({})    // 第 7 行
```

**缓存特点**:
| 特点 | 说明 |
|------|------|
| **存储位置** | Vue 响应式 ref（内存） |
| **生命周期** | 组件卸载时自动释放 |
| **失效机制** | 页面切换时自动清空 |
| **TTL** | 无限制（依赖页面生命周期） |
| **手动清空** | `clearDownstream()`, `clearAllDownstream()` |

**缓存清理方法**:
```javascript
// 清空指定字段的下游选项（第 90-104 行）
function clearDownstream(fieldId) {
  cascadeFields.value.slice(startIndex + 1).forEach(function(downstreamFieldId) {
    options.value[downstreamFieldId] = []
  })
}

// 清空所有下游选项（第 106-113 行）
function clearAllDownstream(fieldId) {
  cascadeFields.value.slice(startIndex).forEach(function(fid) {
    options.value[fid] = []
  })
}
```
