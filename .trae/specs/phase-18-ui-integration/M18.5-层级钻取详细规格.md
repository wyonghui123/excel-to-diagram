## 1. 背景与目标

### 1.1 业务背景

架构数据管理涉及 5 层层级结构：
```
产品 (Product)
  └── 版本 (Version)
        └── 领域 (Domain)
              └── 子领域 (SubDomain)
                    └── 服务模块 (ServiceModule)
                          └── 业务对象 (BusinessObject)
```

用户需要频繁地在不同层级之间钻取（drill-down / drill-up）：
- 从领域列表钻入到子领域
- 从子领域钻入到服务模块
- 从服务模块钻入到业务对象

### 1.2 技术背景

- `hierarchies.yaml` 已定义层级结构
- `HierarchyService` 提供层级数据查询
- `BreadcrumbNav.vue` 已存在，支持基础面包屑

**待完成部分**:
- `useHierarchyList` Composable（钻取状态管理）
- MetaTable 层级增强（路径列、计数列）
- 钻取联动逻辑

### 1.3 目标声明

| 目标类型 | 目标描述 | 成功标准 |
|---------|---------|---------|
| **架构迁移** | 层级钻取能力集成到 MetaTable | 支持点击子对象计数钻入下一层 |
| **通用能力** | 沉淀 useHierarchyList、BreadcrumbNav 增强为通用组件 | 可独立于架构数据管理场景使用 |
| **元数据驱动** | 层级路径、分隔符从 YAML 配置读取 | 无硬编码对象名或层级顺序 |

## 3. 功能需求

### FR-001: 钻取状态管理

- **描述**: 系统 MUST 提供 useHierarchyList Composable 管理钻取状态
- **详细说明**:
  - 管理钻取路径（path）
  - 管理当前对象类型（currentType）
  - 管理父对象 ID（parentId）
  - 提供 drillIn、drillOut、reset 方法
- **验收标准**:
  - [ ] 钻入子对象时 path 增加一个节点
  - [ ] 钻出到父级时 path 移除最后一个节点
  - [ ] 重置时 path 清空到初始状态
  - [ ] isDrilling 状态正确反映钻取进行中
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 架构需求文档

### FR-002: 面包屑导航显示

- **描述**: 系统 MUST 通过 BreadcrumbNav 显示当前钻取路径
- **详细说明**:
  - 复用现有 BreadcrumbNav.vue 组件
  - 支持点击任意层级回退
  - 支持对象类型图标显示
  - 支持路径过长时的省略显示
- **验收标准**:
  - [ ] 路径显示格式：领域名称 > 子领域名称 > 服务模块名称
  - [ ] 点击面包屑任意节点正确回退到该层级
  - [ ] 对象类型图标正确显示
  - [ ] 超过 4 级时显示省略号
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: FR-001 的实现需求

### FR-003: 层级路径列渲染

- **描述**: MetaTable MUST 支持显示层级路径列
- **详细说明**:
  - 显示从根节点到当前节点的完整路径
  - 分隔符可配置（默认 ' > '）
  - 路径从 YAML hierarchies 配置动态构建
- **验收标准**:
  - [ ] Domain 列表显示完整路径（仅当前节点）
  - [ ] SubDomain 列表显示 "领域 > 子领域"
  - [ ] ServiceModule 列表显示 "领域 > 子领域 > 服务模块"
  - [ ] 分隔符符合 hierarchies.path_separator 配置
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 用户体验需求

### FR-004: 子对象计数列

- **描述**: MetaTable MUST 支持显示子对象计数列
- **详细说明**:
  - 显示每个节点下的子对象数量
  - 支持点击计数钻入子对象列表
  - 计数通过 API 查询获取
- **验收标准**:
  - [ ] Domain 列表显示 "X 个子领域"
  - [ ] SubDomain 列表显示 "X 个服务模块"
  - [ ] 点击计数文本触发 drillIn
  - [ ] 无子对象时显示 "无" 或 "-"
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 核心功能需求

### FR-005: 钻取联动逻辑

- **描述**: 钻取操作 MUST 触发正确的列表刷新和状态更新
- **详细说明**:
  - 钻入时：更新 currentType + parentId + 刷新列表 + 更新面包屑
  - 钻出时：回退到指定层级 + 刷新列表 + 更新面包屑
  - 列表带上 parent_id 过滤条件
- **验收标准**:
  - [ ] drillIn 后列表数据正确过滤
  - [ ] drillOut 后列表数据正确回退
  - [ ] 面包屑状态同步更新
  - [ ] URL 参数同步更新（如支持）
- **优先级**: Must
- **类型映射**: 功能需求
- **来源**: 核心功能需求

## 4. 非功能需求

### NFR-001: 性能要求

- **描述**: 钻取操作响应时间应在合理范围内
- **测量方法**:
  - 钻取操作到列表刷新：< 500ms
  - 子对象计数加载（批量）：< 1s
  - 面包屑更新：< 50ms
- **优先级**: Should
- **来源**: 用户体验要求

### NFR-002: 可用性要求

- **描述**: 钻取交互应直观易懂
- **测量方法**:
  - 计数列悬停提示明确
  - 钻取操作有视觉反馈
  - 面包屑点击区域足够大
- **优先级**: Must
- **来源**: 用户体验要求

### NFR-003: 可访问性要求

- **描述**: 层级路径应支持屏幕阅读器
- **测量方法**:
  - 面包屑有 aria-label
  - 计数列有 title 属性
  - 键盘可操作钻取
- **优先级**: Should
- **来源**: 无障碍访问要求

  ```typescript
  interface CountResponse {
    success: boolean
    data: {
      count: number
    }
    message?: string
  }
  ```

- **错误处理**:
  - 400: 参数错误
  - 404: 对象类型不存在
  - 500: 服务器错误

### 8.3 详细设计

#### 8.3.1 useHierarchyList Composable 设计

```javascript
// src/composables/useHierarchyList.js

import { ref, computed, watch } from 'vue'
import { boService } from '@/services/boService'

/**
 * [COMPOSABLE] 层级钻取组合函数
 * [DESCRIPTION] 管理面包屑导航和钻取状态，基于 YAML hierarchies 配置
 * 
 * @param {Object} options - 配置选项
 * @returns {Object} 钻取控制接口
 */
export function useHierarchyList(options = {}) {
  const {
    versionId = ref(null),
    initialType = 'domain',
    onPathChange = null
  } = options

  // ============================================================
  // [SECTION] 响应式状态
  // ============================================================
  
  const path = ref([])
  const isDrilling = ref(false)
  const loading = ref(false)

  // 当前对象类型（从路径最后一个节点获取）
  const currentType = computed(() => {
    return path.value.length > 0 
      ? path.value[path.value.length - 1].type 
      : initialType
  })

  // 父对象 ID（从路径最后一个节点获取）
  const parentId = computed(() => {
    return path.value.length > 0 
      ? path.value[path.value.length - 1].id 
      : null
  })

  // 是否有父级可返回
  const canDrillOut = computed(() => path.value.length > 0)

  // ============================================================
  // [SECTION] 核心方法
  // ============================================================
  
  /**
   * [METHOD] drillIn
   * [DESCRIPTION] 钻入到子对象
   * 
   * @param {string} targetType - 目标对象类型
   * @param {number} parentId - 父对象 ID
   * @param {string} parentName - 父对象名称
   * @returns {Promise<void>}
   */
  async function drillIn(targetType, parentId, parentName) {
    isDrilling.value = true
    loading.value = true
    
    try {
      const newNode = {
        type: targetType,
        id: parentId,
        name: parentName
      }
      
      path.value = [...path.value, newNode]
      
      // 触发回调
      if (onPathChange) {
        onPathChange(path.value)
      }
    } finally {
      isDrilling.value = false
      loading.value = false
    }
  }

  /**
   * [METHOD] drillOut
   * [DESCRIPTION] 钻出到指定层级
   * 
   * @param {number} index - 目标层级索引
   * @returns {void}
   */
  function drillOut(index) {
    if (index < 0 || index >= path.value.length) return
    
    path.value = path.value.slice(0, index + 1)
    
    // 触发回调
    if (onPathChange) {
      onPathChange(path.value)
    }
  }

  /**
   * [METHOD] reset
   * [DESCRIPTION] 重置到初始状态
   * 
   * @returns {void}
   */
  function reset() {
    path.value = []
    
    // 触发回调
    if (onPathChange) {
      onPathChange(path.value)
    }
  }

  /**
   * [METHOD] goTo
   * [DESCRIPTION] 跳转到指定层级（别名方法，与 BreadcrumbNav 兼容）
   * 
   * @param {number} index - 目标层级索引
   * @returns {void}
   */
  function goTo(index) {
    drillOut(index)
  }

  /**
   * [METHOD] getChildObjectType
   * [DESCRIPTION] 获取子对象类型
   * 
   * @param {string} currentType - 当前对象类型
   * @returns {string|null} 子对象类型
   */
  function getChildObjectType(currentType) {
    const typeOrder = ['product', 'version', 'domain', 'sub_domain', 'service_module', 'business_object']
    const currentIndex = typeOrder.indexOf(currentType)
    
    if (currentIndex === -1 || currentIndex >= typeOrder.length - 1) {
      return null
    }
    
    return typeOrder[currentIndex + 1]
  }

  /**
   * [METHOD] getParentObjectType
   * [DESCRIPTION] 获取父对象类型
   * 
   * @param {string} currentType - 当前对象类型
   * @returns {string|null} 父对象类型
   */
  function getParentObjectType(currentType) {
    const typeOrder = ['product', 'version', 'domain', 'sub_domain', 'service_module', 'business_object']
    const currentIndex = typeOrder.indexOf(currentType)
    
    if (currentIndex <= 0) {
      return null
    }
    
    return typeOrder[currentIndex - 1]
  }

  // ============================================================
  // [SECTION] 导出接口
  // ============================================================
  
  return {
    // 状态
    path,
    currentType,
    parentId,
    isDrilling,
    canDrillOut,
    loading,
    
    // 方法
    drillIn,
    drillOut,
    reset,
    goTo,
    getChildObjectType,
    getParentObjectType
  }
}
```

### 8.4 替代方案考虑

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| **方案 A**: useHierarchyList Composable | 状态集中，易测试 | 需要学习新 API | **已选** |
| 方案 B: Pinia Store | 全局状态共享 | 过度设计，小功能不需要 | 备选 |
