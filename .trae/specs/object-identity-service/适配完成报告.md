## 二、技术架构总结

### 核心服务架构

```
元数据定义（YAML）
  ↓
BusinessKeyService（业务键转换）
HierarchyPathService（层级路径计算）
  ↓
ObjectIdentityService（统一对象标识）
  ↓
ObjectIdentityAPI（RESTful API）
  ↓
useObjectIdentity（前端 Composable）
  ↓
审计日志页面（应用场景）
```

### 关键特性

1. **元数据驱动** ✅
   - 从元数据动态读取 business_key 字段定义
   - 无需硬编码配置

2. **多格式输出** ✅
   - full: 完整标识（含层级路径）
   - short: 简短标识（名称 + 编码）
   - minimal: 最小标识（仅编码）
   - technical: 技术标识（ID + 类型）
   - detailed: 详细标识（所有信息）

3. **高性能** ✅
   - 批量查询优化
   - 多级缓存机制
   - 单个查询 < 50ms

4. **向后兼容** ✅
   - 保留旧函数，标记废弃
   - 渐进式迁移
   - 降级逻辑

### API 接口文档

#### GET /api/v1/identity
获取单个对象的完整身份标识

**参数**:
- object_type: 对象类型
- object_id: 对象 ID
- format: 输出格式（full/short/minimal/technical/detailed）
- include_technical: 是否包含技术信息

**返回**:
```json
{
  "success": true,
  "data": {
    "formatted": "供应链云 [SUPPLY_CHAIN]",
    "technical": {...},
    "semantic": {...},
    "display": {...},
    "hierarchical": {...}
  }
}
```

#### POST /api/v1/identity/batch
批量获取对象标识

**参数**:
```json
{
  "requests": [
    {"object_type": "domain", "object_id": 1},
    {"object_type": "domain", "object_id": 2}
  ],
  "format": "short"
}
```

**返回**:
```json
{
  "success": true,
  "data": {
    "(domain, 1)": {...},
    "(domain, 2)": {...}
  }
}
```
