# 图表备注功能需求规范

## Why

用户需要在 Excel 转 Diagram 生成的架构图中，为核心要素（容器、节点、关系）添加补充说明，以便更好地传达业务语义和关键信息。当前系统缺乏这类标注能力，用户期望在不干扰主图布局的前提下，实现类似学术论文的引用标注效果。

## What Changes

### 功能概述

在现有业务对象图和服务模块图的基础上，新增**备注功能模块**，允许用户为图表中的容器、节点、关系添加带编号的备注说明，并通过独立的备注区域集中展示。

### 核心特性

1. **备注添加**

   * Excel 数据源增加备注列（备注分类、备注内容）

   * 支持为业务对象、服务模块、业务对象关系添加备注

   * 备注数量建议：10容器+30节点 → 约 8 个备注

2. **编号标识**

   * 主图要素显示圆圈数字编号（①②③...）

   * 编号通过 SVG 叠加层实现，不影响主图原有布局

   * 备注区域编号与主图编号一一对应

3. **分类标签**

   * 四种分类：重要（⚠️）、警告（🚨）、信息（ℹ️）、提示（💡）

   * 不同分类有独立的颜色和样式区分

4. **备注位置控制**

   * 在配置入口中设置备注面板位置

   * 支持三种位置：底部（默认）、右侧、左侧

   * 当备注数量 ≥ 5 时，提示建议选择"底部"位置

5. **交互联动**

   * 点击主图编号 → 高亮对应备注

   * 点击备注项 → 高亮对应主图要素

   * 点击空白处 → 清除高亮状态

## Impact

### 影响的现有规格

* **Excel 数据解析**：businessObject、businessObjectRelationship、serviceComponent 页签增加备注列

* **预览页面**：业务对象列表、服务模块列表、业务对象关系列表展示备注数据

* **图表渲染能力**：新增备注叠加层渲染

* **交互能力**：新增点击高亮交互

### 影响的代码模块

* `src/services/excelParser.js` - 扩展 Excel 备注列解析

* `src/composables/useMermaid/annotation/` - 新增备注功能模块

* `src/components/*List*.vue` - 列表组件展示备注

* `src/components/MermaidComponent.vue` - 集成备注渲染

* `src/styles/` - 新增备注相关样式

## ADDED Requirements

### Requirement: Excel 数据源扩展

系统 SHALL 支持从 Excel 文件中解析备注数据，数据来源为现有页签的增加列：

**businessObject 页签新增列**：

| 列名   | 类型     | 说明                                          |
| ---- | ------ | ------------------------------------------- |
| 备注分类 | enum   | important / warning / info / tip，空值默认为 info |
| 备注内容 | string | 备注说明文字                                      |

**serviceComponent 页签新增列**：

| 列名   | 类型     | 说明                                          |
| ---- | ------ | ------------------------------------------- |
| 备注分类 | enum   | important / warning / info / tip，空值默认为 info |
| 备注内容 | string | 备注说明文字                                      |

**businessObjectRelationship 页签新增列**：

| 列名   | 类型     | 说明                                          |
| ---- | ------ | ------------------------------------------- |
| 备注分类 | enum   | important / warning / info / tip，空值默认为 info |
| 备注内容 | string | 备注说明文字                                      |

### Requirement: 预览列表展示

系统 SHALL 在预览页面的对应列表中展示备注数据：

* **业务对象列表**：每行业务对象显示备注分类图标和备注内容（截取前20字）

* **服务模块列表**：每个服务模块显示备注分类图标和备注内容

* **业务对象关系列表**：每行关系显示备注分类图标和备注内容

### Requirement: 备注配置入口

系统 SHALL 提供备注相关配置：

**备注图标显示**：

* 在图表配置区域提供"显示备注图标"选项

* 选项：是 / 否（默认否）

* 当"否"时，备注面板仅显示编号和内容，不显示分类图标

* 配置保存到图表配置中

### Requirement: 备注面板隐藏/展示切换

系统 SHALL 支持备注面板的隐藏/展示切换：

* 备注面板标题"备注"可点击

* 点击标题切换备注列表的显示/隐藏状态

* 收起状态显示"备注 ▶"，展开状态显示"备注 ▼"

* 收起时面板高度收缩到仅显示标题

### Requirement: 编号叠加渲染

系统 SHALL 在主图渲染完成后，通过 SVG 叠加层为主图要素添加编号标记，要求：

* 编号位置：容器（左上角）、节点（右上角）、关系（标签上方居中）

* 编号样式：圆形背景 + 数字，直径 24px

* 颜色方案：跟随 targetType 类型色

* **不修改** Mermaid 主图语法，保持原有布局不变

### Requirement: 备注面板渲染（方案A：底部Dock模式）

系统 SHALL 采用**底部Dock模式**渲染备注面板，要求：

**布局结构**：

```
┌─────────────────────────────────────────────────────────┐
│                      主图表区域                          │
│                                                          │
│    [1]        [2]                                       │
│  ┌─────┐    ┌─────┐     ┌─────┐                        │
│  │节点A │───→│节点B │     │节点C │                        │
│  └─────┘    └─────┘     └─────┘                        │
│                                                          │
├─────────────────────────────────────────────────────────┤
│  备注面板 (固定高度 150-200px)                           │
│  ┌────────────────────────────────────────────────────┐ │
│  │ ① [info] 节点A的备注内容...                         │ │
│  │ ② [important] 节点B的备注内容...                    │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**面板规格**：

| 属性 | 值 | 说明 |
|------|-----|------|
| 位置 | 图表下方 | 贴近图表，无间隙 |
| 高度 | 固定 150-200px | 不可扩展 |
| 宽度 | 100% | 自适应容器宽度 |
| 溢出处理 | 内容截断 | 超长内容显示省略号 |
| 对齐方式 | 左对齐 | text-align: left |

**内容展示**：

* 每行显示：编号 + 分类图标（可选）+ 备注内容 + 类型标签

* 内容最大行数：2行，超出截断并显示省略号

* 内容最大宽度：根据面板宽度自适应

**滚动控制**：

* **禁止出现滚动条** - 面板高度固定，内容截断

* 如果备注数量过多导致内容无法完整显示，提示用户切换为"右侧"或"左侧"位置

### Requirement: 分类样式

系统 SHALL 为不同分类的备注提供视觉区分：

| 分类        | 图标 | 背景色     | 边框色     | 用途说明       |
| --------- | -- | ------- | ------- | ---------- |
| important | ⚠️ | #fff3cd | #856404 | 重要提醒       |
| warning   | 🚨 | #f8d7da | #721c24 | 警告信息       |
| info      | ℹ️ | #e7f3ff | #0066cc | 一般说明（空值默认） |
| tip       | 💡 | #d4edda | #155724 | 提示建议       |

**备注**：

* 当"备注分类"为空时，默认使用 `info` 分类

* 图标是否显示由配置参数控制（默认不显示）

### Requirement: 交互高亮

系统 SHALL 支持主图编号与备注项的双向交互：

* 点击主图编号 → 高亮对应备注项（背景色加深 + 边框加粗）

* 点击备注项 → 高亮对应主图要素（添加高亮边框）

* 点击空白处 → 清除所有高亮状态

### Requirement: 数据流转

```
Excel 数据
    ↓
excelParser.js 解析
    ↓
businessObject / serviceComponent / businessObjectRelationship 数据结构
    ↓
预览列表组件展示备注
    ↓
diagramDataBuilder 构建图表数据
    ↓
MermaidComponent 渲染主图
    ↓
annotationOverlay 叠加编号和备注面板
```

## User Experience

### 使用流程

1. 用户在 Excel 的 businessObject / serviceComponent / businessObjectRelationship 页签中填写备注列
2. 系统解析 Excel 数据时自动提取备注信息
3. 预览页面列表中显示备注分类图标和内容预览
4. 主图渲染时自动叠加编号标记和备注面板
5. 用户可点击编号/备注查看关联关系
6. 用户可点击备注面板标题切换显示/隐藏状态
