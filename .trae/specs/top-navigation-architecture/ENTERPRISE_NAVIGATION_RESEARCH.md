# 头部企业产品导航系统研究报告

## 研究目的

研究 SAP、Salesforce、Microsoft Dynamics 365 等头部企业产品的导航系统设计，为我们的产品导航优化提供参考。

---

## 一、SAP Fiori 3.0

### 1.1 核心设计理念（已纠正）

**重要澄清**：SAP Fiori Launchpad **不是一个带有传统侧边导航的应用框架**，而是一个**应用入口平台**。

SAP Fiori 的设计哲学：
- 每个业务应用都是**独立的应用**
- Launchpad 是**统一的入口**，展示可访问的应用磁贴
- 用户通过 Launchpad 进入不同的独立应用
- **不在 Launchpad 层面提供全局侧边导航**

### 1.2 Launchpad 页面结构

```
┌──────────────────────────────────────────────────────────────┐
│ [Shell Bar - 顶部固定功能栏]                                  │
│ [Home] [←返回] [页面标题 ▼] | [企业搜索] [Copilot] [通知] [用户] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   Launchpad 主区域 - 磁贴(Tiles)网格布局                       │
│   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐         │
│   │  应用1  │ │  应用2  │ │  应用3  │ │  应用4  │         │
│   │  磁贴   │ │  磁贴   │ │  磁贴   │ │  磁贴   │         │
│   └─────────┘ └─────────┘ └─────────┘ └─────────┘         │
│   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐         │
│   │  应用5  │ │  应用6  │ │  应用7  │ │  应用8  │         │
│   │  磁贴   │ │  磁贴   │ │  磁贴   │ │  磁贴   │         │
│   └─────────┘ └─────────┘ └─────────┘ └─────────┘         │
│                                                              │
│   [最近使用] [常用应用]                                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 1.3 Shell Bar 功能（已纠正）

Shell Bar **不是传统导航栏**，而是提供全局功能的快捷入口栏：

| 组件 | 说明 |
|------|------|
| **Home** | 返回 Launchpad 首页 |
| **Back** | 返回上一页（应用内导航时显示） |
| **Page Title ▼** | 当前应用名称 + 下拉菜单（显示**应用内**导航层级） |
| **Enterprise Search** | 跨应用全局搜索 |
| **Copilot (Joule)** | AI 智能助手按钮 |
| **Notifications** | 通知中心铃铛 |
| **User Menu** | 用户头像，点击打开 Me Area |

**关键澄清**：
- Page Title 的下拉菜单**只显示当前应用内的导航层级**
- **不是跨应用的全局导航**
- 这是 SAP Fiori 与 Salesforce/Dynamics 365 的核心区别

### 1.4 Me Area（已纠正）

Me Area **不是固定侧边栏**，而是点击用户头像时从**左侧滑出的面板**：

```
点击用户头像前：
┌──────────────────────────────────────┐
│                           [用户头像] │
└──────────────────────────────────────┘

点击用户头像后（Me Area 滑出）：
┌────────────────┬─────────────────────┐
│   Me Area      │                     │
│ ┌────────────┐ │                     │
│ │ 用户信息   │ │                     │
│ │ 最近访问   │ │   Launchpad 主区域  │
│ │ 常用应用   │ │                     │
│ │ 设置       │ │                     │
│ │ 退出       │ │                     │
│ └────────────┘ │                     │
└────────────────┴─────────────────────┘
```

### 1.5 应用内导航（已纠正）

进入具体应用后，导航在**应用内部**实现：

- 使用 **Flexible Column Layout**（灵活列布局）
- 可以有 **Tab 页**
- 可以有**应用内的侧边导航**（但属于单个应用，不是全局框架）
- 使用 **Object Page** 等页面模式

### 1.6 与其他产品的对比

| 特性 | SAP Fiori | Salesforce | Dynamics 365 |
|------|-----------|-----------|--------------|
| **入口模式** | 磁贴网格 | 固定导航栏 | 固定侧边栏 |
| **全局导航** | ❌ 无全局导航 | ✅ 顶部导航栏 | ✅ 左侧站点地图 |
| **应用切换** | Launchpad 磁贴 | App Launcher | App Selector |
| **侧边栏** | Me Area (滑出式) | 固定侧边栏 | 可折叠侧边栏 |
| **哲学** | 独立应用集合 | 单一应用框架 | 单一应用框架 |

## 二、Salesforce Lightning Experience

### 2.1 导航架构

Salesforce Lightning 采用传统的**单一应用框架**：

```
┌──────────────────────────────────────────────────────────────┐
│ [App Logo] [App Name ▼] │ [对象1] [对象2] [对象3] [更多▼] │
│                                                [搜索] [帮助] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   主内容区域                                                   │
│                                                              │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  侧边栏 (可折叠)                                     │  │
│   │  - 最近记录                                          │  │
│   │  - 收藏夹                                            │  │
│   │  - 对象列表                                          │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ [实用工具栏 - 固定底部]                                       │
│ [笔记] [聊天] [记录] [电话] [更多...]                         │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 核心组件

| 组件 | 说明 |
|------|------|
| **导航栏** | 顶部水平导航，每个应用可配置不同 |
| **侧边栏** | 可折叠，显示最近记录和收藏夹 |
| **App Launcher** | 网格化应用切换界面 |
| **实用工具栏** | 底部固定工具栏 |

## 四、设计模式对比分析

### 4.1 入口模式对比

| 产品 | 入口模式 | 说明 |
|------|---------|------|
| **SAP Fiori** | **Launchpad 磁贴** | 独立的业务应用集合，通过入口页面访问 |
| **Salesforce** | **固定导航栏** | 单一应用框架，顶部水平导航 |
| **Dynamics 365** | **固定侧边栏** | 单一应用框架，左侧垂直导航 |
| **我们的方案** | **混合模式** | 顶部导航 + 可折叠侧边栏 |

### 4.2 导航哲学对比

| 产品 | 导航哲学 | 特点 |
|------|---------|------|
| **SAP Fiori** | **独立应用集合** | 每个业务场景是独立应用，无全局导航 |
| **Salesforce** | **单一应用框架** | 所有功能在单一应用内，全局导航栏 |
| **Dynamics 365** | **单一应用框架** | 同 Salesforce，左侧站点地图 |
| **我们的方案** | **混合模式** | 类似 Salesforce + 增强功能 |

### 4.3 侧边栏设计对比

| 产品 | 侧边栏类型 | 是否固定 | 功能 |
|------|-----------|---------|------|
| **SAP Fiori** | Me Area (滑出式) | ❌ 按需滑出 | 用户信息、最近访问、设置 |
| **Salesforce** | 固定侧边栏 | ✅ 可折叠 | 最近记录、收藏夹、对象列表 |
| **Dynamics 365** | 固定 Site Map | ✅ 可折叠 | 最近记录、收藏夹、实体导航 |
| **我们的方案** | 固定侧边栏 | ✅ 可折叠 | 菜单导航、收藏夹、最近访问 |

## 五、我们的方案定位

### 5.1 设计选择

我们的方案采用**混合模式**，更接近 Salesforce 和 Dynamics 365：

```
┌──────────────────────────────────────────────────────────────┐
│ [TopNavHeader]                                              │
│ [Logo] [面包屑] │ [全局搜索] │ [通知] [用户菜单]            │
├──────────────────────────────────────────────────────────────┤
│ [AppTabs - 多页面 Tab 栏]                                    │
├────────────────┬─────────────────────────────────────────────┤
│                │                                             │
│ AppSideNav    │   主内容区域                                 │
│ (可折叠)      │                                             │
│                │                                             │
│ - 菜单导航    │                                             │
│ - 收藏夹      │                                             │
│ - 最近访问    │                                             │
│                │                                             │
└────────────────┴─────────────────────────────────────────────┘
```

### 5.2 与 SAP Fiori 的核心区别

| 特性 | SAP Fiori | 我们的方案 |
|------|-----------|-----------|
| **应用模式** | 独立应用集合 | 单一应用框架 |
| **全局导航** | 无（只有 Launchpad 入口） | ✅ 有（顶部 + 侧边） |
| **多页面 Tab** | ❌ 无 | ✅ 有 |
| **侧边栏** | Me Area (滑出式) | 固定侧边栏 |
| **适用场景** | 多系统集成 | 单一复杂应用 |

### 5.3 适用场景分析

**选择 SAP Fiori 模式**：
- 多套独立系统需要统一入口
- 各系统之间相对独立
- 需要跨系统全局搜索和通知

**选择我们的方案（混合模式）**：
- 单一复杂业务应用
- 需要在应用内多页面切换
- 用户需要同时操作多个功能模块
- 需要持久化的 Tab 状态

## 七、总结

### 7.1 核心发现（已纠正）

1. **SAP Fiori 采用不同的导航哲学**：是独立应用的集合，不是在单一应用内提供全局导航
2. **Shell Bar 不是传统导航栏**：是功能快捷入口，Page Title 下拉菜单只显示应用内导航
3. **Me Area 是滑出式面板**：不是固定侧边栏
4. **我们的方案更接近 Salesforce/Dynamics 365**：采用单一应用框架 + 全局导航

### 7.2 方案定位

我们的方案选择**混合模式**是正确的：
- 适合单一复杂业务应用
- 需要多页面 Tab 管理
- 需要持久化的用户状态
- 需要灵活的内容区域

### 7.3 后续优化方向

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 侧边栏折叠 | 默认折叠，释放空间 |
| P1 | 收藏夹 | 快速访问常用功能 |
| P1 | 实用工具栏 | 固定工具快速访问 |
| P2 | 最近访问 | 记录最近操作的记录/页面 |
| P2 | 全局搜索 | 跨模块搜索功能 |
