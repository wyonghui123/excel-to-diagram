## 🔍 详细失败原因分析

### 1. 审计日志问题 (13个失败)

**根本原因**: API 路径问题
- 测试使用 `/api/v2/bo/audit_log` 但实际路径可能不同
- 登录状态验证失败导致后续请求返回 401

**典型错误**:
```
Error: expect(received).toBeTruthy()
Received: false
at annotation-metadata.spec.js:39:32
```

**建议修复**:
- 检查审计日志 API 的正确路径
- 确保测试前正确登录并获取 token

### 2. 弹窗/对话框问题 (10个失败)

**根本原因**: 登录遮罩层阻挡点击
- `login-overlay` 元素遮挡了用户菜单触发器
- 测试尝试点击 `.user-menu__trigger` 但被遮罩层拦截

**典型错误**:
```
TimeoutError: locator.click: Timeout 15000ms exceeded.
<div class="login-overlay">...</div> intercepts pointer events
```

**建议修复**:
- 在打开弹窗前确保登录状态已完全加载
- 增加等待登录遮罩层消失的逻辑

### 3. 备注/Annotation问题 (8个失败)

**根本原因**: UI Config 配置问题
- `child_section` 配置未正确返回 annotation 面板
- 详情页未显示备注面板

**建议修复**:
- 检查 YAML 配置中的 `child_sections` 定义
- 验证 UI Config API 返回的数据结构

### 4. 关联关系问题 (6个失败)

**根本原因**: 详情页关联面板未渲染
- 用户详情页未显示"所属用户组"面板
- 角色详情页未显示"用户"和"用户组"面板

**建议修复**:
- 检查关联配置是否正确加载
- 验证详情页组件是否正确渲染关联面板
