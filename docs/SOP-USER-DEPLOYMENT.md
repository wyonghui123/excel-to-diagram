# 用户傻瓜化部署 SOP

> 版本: v1.1
> 更新日期: 2026-04-29
> 状态: 生效中

---

## 一、核心原则

**用户只需要说 "开始部署"，其他全部自动化！**

```
用户操作：说 "开始部署" → 复制 → 粘贴 → 完成！
```

**自动化能力：**
- ✅ 部署打包
- ✅ 上传
- ✅ 部署执行
- ✅ 自动测试验证（ci-cd-test.sh）
- ✅ 健康检查（health-check.sh）
- ✅ 生成部署报告

---

## 十、代码开发规范（重要 ⚠️）

### 10.1 新建 API 必须遵守

创建新 API 文件时，**必须**使用以下模板：
```
meta/api/API_TEMPLATE.py
```

### 10.2 数据库路径规范

```python
# ✅ 正确
import os
db_path = os.path.join(
    os.path.dirname(os.path.dirname(os.path.abspath(__file__))),
    'architecture.db'
)
_data_source = get_data_source("sqlite", database=db_path)

# ❌ 禁止使用相对路径
_data_source = get_data_source("sqlite", database="meta/architecture.db")
```

### 10.3 创建新 API 后

1. 复制模板：`meta/api/API_TEMPLATE.py`
2. 在 `server.py` 注册蓝图
3. 提交代码时确保无相对路径

## 十一、附录：常用命令速查

### 服务器端命令

```bash
# 查看服务状态
netstat -tlnp | grep -E "8081|5001"

# 查看日志
tail -f /opt/app/shared/logs/deploy.log
tail -f /opt/app/shared/logs/backend.log

# 重启服务
pkill -f "server.py"; sleep 2
cd /opt/app && python server.py &
cd /opt/app/meta && python server.py &

# 回滚
cd /opt/app
PREV=$(cat /opt/app/state/previous_version)
ln -sfn /opt/app/backups/$PREV /opt/app/current
pkill -f "server.py"; sleep 2
cd /opt/app/current && python server.py &
```
