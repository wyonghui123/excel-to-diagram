# Spec: 权限体系元数据驱动化升级

> 日期：2026-05-16 | 版本：v1.0 | 状态：待确认

## 1. 背景与目标

### 1.1 背景

当前项目基于 YAML + BO Framework 的元数据驱动架构运行良好，但权限体系存在 **10处** 违反单一事实源原则的硬编码断裂点。五家头部产品（SAP/Salesforce/PowerPlatform/ServiceNow/用友BIP）的对比研究表明：组织/维度应作为权限配置的第一入口。

### 1.2 业务目标

- BO-01: 元数据成为唯一事实源——新增BO零代码改动即可获得完整权限体系
- BO-02: 权限配置入口从"菜单"改为"管理维度"——符合业务语言和行业最佳实践
- BO-03: 消除10处硬编码映射字典——统一从 MetaRegistry 动态获取

### 1.3 涉众目标

- SG-01: 管理员可用维度声明方式快速配置权限，减少70%配置步骤
- SG-02: 开发者新增BO时不再需要修改 init_auth.py / init_menu_permissions.py / 前端 menuConfig.js
- SG-03: 普通用户获得与组织范围精准匹配的菜单和操作权限
