# Excel to Diagram 需求文档

## 1. 项目概述

### 1.1 项目简介

| 属性 | 值 |
|------|-----|
| **项目名称** | ArchWorkspace / excel-to-diagram |
| **项目类型** | 架构可视化工具 |
| **核心功能** | 将 Excel/CSV 数据转换为架构图、流程图，支持导出多种格式 |
| **目标用户** | 架构师、开发团队、技术管理者 |

### 1.2 核心价值

- **降低门槛**：让非技术人员也能快速生成专业架构图
- **保持一致**：基于统一数据源，确保文档与代码一致性
- **提升效率**：自动化繁琐的图表绘制工作

### 1.3 约束与限制

- 纯前端应用，无需后端（除非使用飞书集成）
- 支持桌面端（Electron）和浏览器端
- 国内访问需要考虑网络问题

## 2. 用户角色

### 2.1 主要角色

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| **架构师** | 设计系统架构，生成架构文档 | 上传Excel，生成应用架构图 |
| **开发人员** | 理解系统结构，查看模块关系 | 查看服务模块之间的依赖 |
| **技术管理者** | 了解团队技术栈，规划项目 | 查看业务对象关系 |

### 2.2 用户故事总览

```
作为一名 架构师
我想要 上传包含业务对象和关系的Excel文件
以便 自动生成可视化的架构图

作为一名 开发人员
我想要 选择特定的数据范围进行展示
以便 聚焦在我关心的模块上

作为一名 技术管理者
我想要 导出高质量的架构图用于文档
以便 在汇报中使用
```

### 3.2 数据导入

#### 3.2.1 Excel文件解析

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 文件选择 | 支持选择本地Excel/CSV文件 | P0 |
| 文件拖拽 | 支持拖拽文件到上传区域 | P1 |
| 数据预览 | 解析后展示数据预览 | P0 |
| 格式校验 | 检查文件格式和内容是否有效 | P0 |
| 多Sheet支持 | 支持读取Excel中的多个Sheet | P1 |

#### 3.2.2 数据结构要求

Excel文件应包含以下 Sheet：

| Sheet名称 | 必填 | 描述 |
|-----------|------|------|
| 领域产品 | 是 | 领域(Domain)和子领域(SubDomain)定义 |
| 业务对象 | 是 | 业务对象(BO)定义，包含编码、名称、所属子领域 |
| 服务模块 | 是 | 服务模块定义，包含与BO的关系 |
| 关系 | 是 | 业务对象之间的关系 |

#### 3.2.3 数据字段

**领域产品 (DomainProducts)**

| 字段 | 类型 | 描述 |
|------|------|------|
| 领域编码 | string | 领域唯一标识 |
| 领域名称 | string | 领域显示名称 |
| 子领域编码 | string | 子领域唯一标识 |
| 子领域名称 | string | 子领域显示名称 |

**业务对象 (BusinessObjects)**

| 字段 | 类型 | 描述 |
|------|------|------|
| 对象编码 | string | 业务对象唯一标识 |
| 对象名称 | string | 业务对象显示名称 |
| 子领域 | string | 所属子领域 |
| 说明 | string | 业务对象描述 |

**服务模块 (ServiceModules)**

| 字段 | 类型 | 描述 |
|------|------|------|
| 模块编码 | string | 模块唯一标识 |
| 模块名称 | string | 模块显示名称 |
| 子领域 | string | 所属子领域 |
| 说明 | string | 模块描述 |

**关系 (Relations)**

| 字段 | 类型 | 描述 |
|------|------|------|
| 源对象编码 | string | 关系源业务对象 |
| 目标对象编码 | string | 关系目标业务对象 |
| 关系类型 | string | 关系类型（如：依赖、调用、包含） |
| 关系说明 | string | 关系的详细描述 |

## 5. 数据流

### 5.1 核心数据流

```
Excel文件
    │
    ▼
┌─────────┐     ┌─────────────┐     ┌────────────┐
│ SheetJS │────▶│  数据预览    │────▶│  范围选择   │
│ 解析器   │     │             │     │            │
└─────────┘     └─────────────┘     └────────────┘
                                            │
                                            ▼
┌─────────┐     ┌─────────────┐     ┌────────────┐
│ 导出    │◀────│  Mermaid    │◀────│  配置选择   │
│ 格式    │     │  渲染器      │     │            │
└─────────┘     └─────────────┘     └────────────┘
```

### 5.2 数据模型

#### 5.2.1 核心实体

```javascript
// 领域
Domain {
  code: string,        // 领域编码
  name: string,        // 领域名称
  subDomains: SubDomain[]
}

// 子领域
SubDomain {
  code: string,        // 子领域编码
  name: string,        // 子领域名称
  domainCode: string   // 所属领域编码
}

// 业务对象
BusinessObject {
  code: string,        // 对象编码
  name: string,        // 对象名称
  subDomainCode: string, // 所属子领域
  description: string  // 描述
}

// 服务模块
ServiceModule {
  code: string,        // 模块编码
  name: string,        // 模块名称
  subDomainCode: string, // 所属子领域
  boCodes: string[],   // 包含的业务对象编码
  description: string  // 描述
}

// 关系
Relation {
  sourceCode: string,  // 源对象编码
  targetCode: string,  // 目标对象编码
  type: string,        // 关系类型
  description: string  // 关系说明
}
```

### 5.3 模块职责

| 模块 | 职责 |
|------|------|
| excelParser | Excel文件解析，数据提取 |

| MermaidGenerator | 生成Mermaid语法代码 |

## 8. 业务规则

### 8.1 数据校验规则

#### 8.1.1 校验级别

| 级别 | 说明 | 影响 |
|------|------|------|
| ERROR | 严重错误 | 阻止图表生成，必须修复 |
| WARNING | 警告 | 可能影响展示，建议修复 |
| INFO | 提示 | 信息性建议 |

#### 8.1.2 校验类型

| 类型 | 说明 | 触发条件 |
|------|------|----------|
| FOREIGN_KEY | 外键关联 | 引用的实体不存在 |
| REQUIRED | 必填项 | 必填字段为空 |
| DUPLICATE | 重复数据 | 存在重复的编码 |
| FORMAT | 格式错误 | 数据格式不符合要求 |
| AI_CHECK | AI检查 | AI识别出的问题 |

#### 8.1.3 外键校验规则

```
业务对象 → 服务模块：引用的服务模块编码必须存在
关系 → 业务对象：源/目标业务对象编码必须存在
服务模块 → 领域：引用的领域编码建议存在（可选）
```

#### 8.1.4 自关联检测

- 关系中源业务对象和目标业务对象相同 → WARNING
- 需用户确认是否为预期行为

### 8.3 布局引擎规则

#### 8.3.1 Dagre 与 ELK 方向映射

| 用户设置方向 | Dagre rankdir | ELK direction |
|--------------|----------------|---------------|
| TB (上下) | TB | LR (左右) |
| LR (左右) | LR | TB (上下) |

**原理**：ELK 的方向与 Dagre 相反，需要反转以保持用户预期的视觉效果。

#### 8.3.2 布局引擎参数

**Dagre 参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| nodeSpacing | 80-150 | 节点间距 |
| rankSpacing | 80-200 | 层级间距 |
| padding | 25-30 | 图表内边距 |
| wrappingWidth | 200-500 | 文字换行宽度 |

**ELK 参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| elk.spacing.nodeNode | 100 | 节点间距 |
| elk.layered.spacing.nodeNodeBetweenLayers | 150 | 层级间节点间距 |
| elk.padding | [top=40,left=50,right=50,bottom=40] | 内边距 |
| elk.hierarchyHandling | INCLUDE_CHILDREN | 层级处理方式 |
| elk.aspectRatio | 1.6 | 宽高比 |

#### 8.3.3 自动布局计算规则

```javascript
// 基于节点数量的布局策略
if (containerCount <= 3) {
  rankdir = 'LR'      // 少量节点用水平布局
  nodeSpacing = 120    // 较大节点间距
  aspectRatio = 1.2
} else if (containerCount <= 6) {
  rankdir = 'TB'      // 中等数量用垂直布局
  nodeSpacing = 100
  aspectRatio = 0.8
} else {
  rankdir = 'TB'      // 大量节点用垂直布局
  nodeSpacing = 100
  aspectRatio = 0.8
}

// 基于连接密度的调整
if (densityRatio > 2) {        // 连接数/节点数 > 2
  rankSpacing *= 0.85          // 减小层级间距
  nodeSpacing *= 0.9           // 减小节点间距
}

// 基于容器内节点数的调整
if (avgNodesPerContainer > 5) {
  nodeSpacing *= 1.15          // 增加节点间距
}
```

### 8.5 节点尺寸计算规则

#### 8.5.1 节点宽度计算

```javascript
// 基于内容的宽度计算
textWidth = 最大行字符数 × 字体宽度(24px × 0.65 ≈ 16px)
nodeWidth = max(textWidth + padding × 2, minWidth)
// 默认最小宽度: 200px
```

#### 8.5.2 节点高度计算

```javascript
// 基于内容的高度计算
textHeight = 行数 × 行高(36px)
nodeHeight = max(textHeight + padding × 2, minHeight)
// 默认最小高度: 100px
```
