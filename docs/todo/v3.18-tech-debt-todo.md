### TODO-1: `tabFiltersVersion` 跨模块响应式根因修复

**问题描述：**
`useMultiObjectPage.js` 的 `tabFiltersVersion` ref 是反模式的 workaround。

```js
const tabFiltersVersion = ref(0)
const tabFilters = computed(() => {
  void tabFiltersVersion.value  // ← 手动 hack 触发依赖
  ...
})
```

注释说"Vue 3 跨模块 reactive proxy 追踪会丢失"。这是 Vue 3 已知行为：`buildRelationshipFilterParams` 接收 `scopeIds.relationExtra`（reactive proxy）时，proxy 内部的 getter 不会被外层 computed 追踪到。

**当前 workaround：**
`tabFiltersVersion` ref + watch + `void tabFiltersVersion.value` 触发追踪。

**真正 fix（待验证）：**
- 方案 A：在 `buildRelationshipFilterParams` 接收 `toRaw(relationExtra)` 纯对象
- 方案 B：让 `buildRelationshipFilterParams` 是 reactive 友好的纯函数（只读 plain object）
- 方案 C：改 `scopeIds.relationExtra` 用 `shallowReactive`（避免深层 proxy）

**风险评估：**
涉及响应式链路核心，涉及面大，需要充分测试。暂用 workaround，等 Vue 3 生态稳定后再重构。

**相关文件：**
- `src/composables/useMultiObjectPage.js`（`tabFiltersVersion`、`tabFilters` computed、`watch`）
- `src/services/hierarchyService.js`（`buildRelationshipFilterParams`、`buildAssociationFilterParams`）
