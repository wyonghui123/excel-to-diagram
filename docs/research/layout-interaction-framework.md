# 布局交互框架分析与建议

## 用户提出的框架

```
1. 是否优化连线（决定了是否用：dagre，elk）
2. 是否控制分组
   2.1 否的话就是默认
   2.2 是的话用户自己来分组，分组就是目前我们的控制布局和嵌套分组之类的逻辑
      2.2.1 是否控制顺序（默认不控制）：是的话才需要选择方向
```

## 当前布局类型分析

### 现有布局选项

| 布局类型 | 名称 | 引擎 | 描述 |
|---------|------|------|------|
| DEFAULT | 自动布局 | dagre | 自动决定最佳布局 |
| HORIZONTAL | 水平排列 | elk | 单行水平展开 |
| VERTICAL | 垂直排列 | elk | 单列垂直展开 |
| ZONE | 分区布局 | elk | 上中下分区，容器均匀分布 |
| GROUPED | 布局控制 | dagre/elk | 自定义分组布局 |

### 问题分析

#### 问题1：水平排列、垂直排列与分组控制的关系

**现状：**
- 水平排列、垂直排列是独立的布局选项
- 分组控制（GROUPED）也有方向控制

**问题：**
- 功能重叠：水平排列 = 单一分组 + 水平方向
- 用户困惑：什么时候用水平排列，什么时候用分组控制？

#### 问题2：分区布局的定位

**现状：**
- 分区布局是特殊的网格布局
- 与分组控制有部分功能重叠

**问题：**
- 分区布局是否应该保留？
- 如果保留，它与分组控制的关系是什么？

#### 问题3：连线优化的理解

**用户理解：**
- dagre：优化连线
- elk：不优化连线

**实际情况：**
- dagre 和 elk 都是布局引擎，都会优化连线
- 区别在于：
  - dagre：传统有向图布局，连线优化优先
  - elk：更现代的布局引擎，支持更复杂的布局需求，对分组方向控制更好

## 完善后的交互框架

### 核心维度

```
维度1：布局引擎选择
├── dagre（传统布局引擎）
│   ├── 优点：连线优化好，布局紧凑
│   ├── 缺点：分组方向控制较弱
│   └── 适用：连线关系复杂，需要优化连线的场景
│
└── elk（现代布局引擎）
    ├── 优点：支持复杂布局，分组方向控制好
    ├── 缺点：布局可能不如 dagre 紧凑
    └── 适用：需要精确控制分组方向的场景

维度2：分组控制
├── 否：自动布局（DEFAULT）
│   └── 让 Mermaid 自动决定布局
│
└── 是：自定义布局
    ├── 简单模式（水平/垂直排列）
    │   ├── 水平排列：单一分组 + 水平方向
    │   └── 垂直排列：单一分组 + 垂直方向
    │
    ├── 分区模式（ZONE）
    │   └── 上中下分区，容器均匀分布
    │
    └── 高级模式（GROUPED）
        ├── 是否控制顺序（默认不控制）
        │   ├── 否：让 Mermaid 根据连线方向自动决定
        │   └── 是：用户手动控制分组顺序
        │
        └── 分组方向控制
            ├── 整体方向：TB/LR/BT/RL
            ├── 分组内方向：TB/LR
            └── 容器内方向：TB/LR
```

## 详细分析

### 1. 布局引擎选择（dagre vs elk）

#### 1.1 dagre 引擎

**特点：**
- 传统有向图布局算法
- 优先优化连线，减少连线交叉
- 布局紧凑

**适用场景：**
- 连线关系复杂
- 需要优化连线走向
- 不需要精确控制分组方向

**行为：**
- 根据连线方向自动调整分组位置
- 连线源头分组在上/左

#### 1.2 elk 引擎

**特点：**
- 现代布局引擎
- 支持更复杂的布局需求
- 对分组方向控制更好

**适用场景：**
- 需要精确控制分组方向
- 需要复杂的分组嵌套
- 需要特殊的布局模式（分区、网格）

**行为：**
- 更好地遵循用户定义的分组方向
- 支持分区布局

#### 1.3 选择建议

| 场景 | 推荐引擎 | 原因 |
|-----|---------|------|
| 连线关系复杂，需要优化 | dagre | 连线优化好 |
| 需要精确控制分组方向 | elk | 方向控制好 |
| 需要分区布局 | elk | 支持分区 |
| 不确定 | dagre | 默认选择 |

---

### 2. 分组控制

#### 2.1 自动布局（DEFAULT）

**特点：**
- 不进行任何分组控制
- 让 Mermaid 自动决定布局

**适用场景：**
- 快速查看结果
- 不需要特殊布局要求

**配置：**
- 无需配置

#### 2.2 简单模式（水平/垂直排列）

**特点：**
- 单一分组
- 只控制整体方向

**适用场景：**
- 容器数量较少
- 只需要简单的方向控制

**配置：**
- 选择方向（水平/垂直）

**与高级模式的关系：**
- 水平排列 = 高级模式 + 单一分组 + 水平方向
- 垂直排列 = 高级模式 + 单一分组 + 垂直方向

#### 2.3 分区模式（ZONE）

**特点：**
- 上中下分区
- 容器均匀分布

**适用场景：**
- 需要将容器分区展示
- 有明确的分区需求

**配置：**
- 分区数量（可选）

**与高级模式的关系：**
- 分区模式是高级模式的特殊应用
- 可以用高级模式实现，但分区模式更简单

#### 2.4 高级模式（GROUPED）

**特点：**
- 完全自定义分组
- 支持嵌套分组
- 支持多层方向控制

**适用场景：**
- 需要精确控制布局
- 有复杂的分组需求

**配置：**
- 分组定义
- 分组顺序控制
- 方向控制（整体/分组内/容器内）

---

### 3. 顺序控制（高级模式特有）

#### 3.1 不控制顺序（默认）

**行为：**
- 让 Mermaid 根据连线方向自动决定分组顺序
- 连线源头分组在上/左

**适用场景：**
- 连线方向有业务意义
- 不需要强制分组顺序

**优点：**
- 简单，无需额外配置
- 连线优化好

**缺点：**
- 无法精确控制分组顺序

#### 3.2 控制顺序

**行为：**
- 用户手动定义分组顺序
- 系统尝试保持用户定义的顺序

**适用场景：**
- 需要精确控制分组顺序
- 分组顺序有业务意义

**配置：**
- 分组顺序（拖拽调整）
- 分组方向

**注意事项：**
- 根据测试结果，连线方向会影响分组顺序
- 如果连线方向与期望顺序冲突，可能无法完全控制

## 重构建议

### 建议1：简化布局选项

**当前：**
```
[自动布局] [水平排列] [垂直排列] [分区布局] [布局控制]
```

**建议：**
```
第一步：选择布局引擎
┌─────────────────────────────────┐
│  ⚡ dagre（优化连线）           │
│  🔧 elk（精确控制）             │
└─────────────────────────────────┘

第二步：选择分组模式
┌─────────────────────────────────┐
│  📦 自动布局（不控制分组）      │
│  ➡️ 水平排列（简单模式）        │
│  ⬇️ 垂直排列（简单模式）        │
│  ⊞ 分区布局（分区模式）         │
│  🎯 布局控制（高级模式）        │
└─────────────────────────────────┘

第三步（仅高级模式）：配置分组
┌─────────────────────────────────┐
│  是否控制顺序：[否] [是]        │
│  如果选择"是"：                 │
│    - 分组顺序（拖拽调整）       │
│    - 整体方向：TB/LR/BT/RL      │
│    - 分组内方向：TB/LR          │
│    - 容器内方向：TB/LR          │
└─────────────────────────────────┘
```

### 建议2：移除冗余选项

**可以移除：**
- 无（所有选项都有其用途）

**可以合并：**
- 水平排列、垂直排列可以作为高级模式的简化入口

### 建议3：保留分区布局

**原因：**
- 分区布局是特殊场景的快捷方式
- 用户可能只需要简单的分区，不需要完整的分组控制

**定位：**
- 分区布局 = 高级模式的快捷方式
- 适用于快速分区需求

## 测试计划

### 测试1：布局引擎对比

**目的：** 验证 dagre 和 elk 在不同场景下的表现

**测试场景：**
1. 简单连线（A→B）
2. 复杂连线（多个分组，多条连线）
3. 分组方向控制

### 测试2：顺序控制

**目的：** 验证"是否控制顺序"的效果

**测试场景：**
1. 不控制顺序 + 单向连线
2. 控制顺序 + 单向连线
3. 不控制顺序 + 双向连线
4. 控制顺序 + 双向连线

### 测试3：分区布局

**目的：** 确定分区布局的保留价值

**测试场景：**
1. 使用分区布局
2. 使用高级模式实现相同效果
3. 对比操作复杂度

## 总结

### 核心发现

1. **布局引擎选择是独立维度**：dagre 和 elk 的选择不应该与分组控制混淆
2. **水平/垂直排列是简化模式**：可以作为高级模式的快捷入口
3. **分区布局有保留价值**：作为特殊场景的快捷方式
4. **顺序控制需要明确选项**：让用户选择是否控制顺序

### 建议的交互框架

```
1. 选择布局引擎（dagre/elk）
2. 选择分组模式
   - 自动布局
   - 水平排列（简化模式）
   - 垂直排列（简化模式）
   - 分区布局（特殊模式）
   - 布局控制（高级模式）
3. 高级模式配置
   - 是否控制顺序
   - 分组定义
   - 方向控制
```
