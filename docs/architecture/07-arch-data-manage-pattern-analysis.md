## 一、现有架构数据管理界面分析

### 1.1 当前界面结构

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│ Header: AppHeader (标题 + 全局 Action: 导入/导出/刷新/展示图表)                       │
├────────────────────────────────────────────────────────────────────────────────────┤
│ ┌─────────────────┐  ┌──────────────────────────────────────────────────────────┐  │
│ │                 │  │ FilterBar (全局过滤)                                    │  │
│ │                 │  ├──────────────────────────────────────────────────────────┤  │
│ │  UnifiedScope   │  │ AppTabs: [对象列表] [关系列表]                          │  │
│ │  Panel         │  ├──────────────────────────────────────────────────────────┤  │
│ │                 │  │ AppTabs (子维度): [领域] [子领域] [服务模块] [业务对象]    │  │
│ │  对象树         │  ├──────────────────────────────────────────────────────────┤  │
│ │  关系树         │  │ Toolbar: [新建] [导出] [删除选中] [搜索...]              │  │
│ │                 │  ├──────────────────────────────────────────────────────────┤  │
│ │  (可调整宽度)    │  │                                                          │  │
│ │                 │  │ DynamicView (列表/详情切换)                                │  │
│ │                 │  │                                                          │  │
│ │                 │  │ - MetaListPage (列表模式)                                │  │
│ │                 │  │ - MetaDetailPage (详情模式)                               │  │
│ │                 │  │                                                          │  │
│ └─────────────────┘  └──────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 当前界面特点

| 区域 | 组件 | 功能 |
|------|------|------|
| Header | AppHeader | 全局导航 + 全局 Action |
| Sidebar | UnifiedScopePanel | 对象树 + 关系树 |
| Main | DynamicView | 列表/详情切换 |
| Tabs | AppTabs | 对象列表/关系列表 + 子维度 |

### 1.3 当前问题

1. **组件职责不清**：UnifiedScopePanel 承担了太多职责（对象树、关系树、维度选择）
2. **缺乏标准 Pattern**：没有参考业界成熟的页面架构
3. **交互复杂**：多层嵌套的 Tab + 多区域联动
4. **复用性差**：组件难以在其他场景复用

#### 2.2.2 Salesforce Console 核心特点

| 特性 | 说明 |
|------|------|
| **Workspace** | 支持打开多个 Primary Tab |
| **Utility Bar** | 底部固定工具栏，快速访问 |
| **Split View** | 左右分栏，可调整宽度 |
| **Related List** | 相关记录列表，支持 inline 编辑 |
| **Activity** | 活动时间线 |

## 四、组件架构设计

### 4.1 推荐的组件层级结构

```
AppWorkspace (顶层容器)
├── WorkspaceHeader (全局 Header)
│   ├── Logo
│   ├── Title
│   ├── GlobalActions (导入/导出/刷新)
│   └── UserMenu
│
├── WorkspaceBody (主内容区)
│   ├── WorkspaceSidebar (可调整宽度)
│   │   ├── DimensionSelector (维度选择)
│   │   ├── ObjectTree (对象树)
│   │   └── RelationTree (关系树)
│   │
│   └── WorkspaceMain (主内容)
│       ├── MainTabs (主 Tab: 对象列表/关系列表)
│       ├── FilterBar (过滤器)
│       ├── Toolbar (工具栏)
│       └── DynamicContent (动态内容)
│           ├── ListView (列表视图)
│           └── DetailPanel (详情面板)
│
└── UtilityBar (可选：底部工具栏)
    ├── QuickFilter
    ├── RecentItems
    └── Help
```

### 4.2 核心组件清单

| 组件 | 说明 | 复用场景 |
|------|------|---------|
| **AppWorkspace** | 工作空间顶层容器 | 所有需要多区域布局的页面 |
| **WorkspaceHeader** | 全局 Header | 所有管理工作区 |
| **WorkspaceSidebar** | 可调整宽度的侧边栏 | 树导航 + 过滤器 |
| **DimensionSelector** | 维度选择器 | 多维度数据管理 |
| **ObjectTree** | 对象树组件 | 层级数据导航 |
| **RelationTree** | 关系类型图 | 关系可视化 |
| **MainTabs** | 主 Tab 导航 | 多视图切换 |
| **FilterBar** | 过滤器栏 | 列表页通用 |
| **Toolbar** | 工具栏 | CRUD 操作 |
| **DetailPanel** | 详情面板 | 侧滑详情 |
| **Drawer** | 抽屉组件 | 表单/详情 |

## 五、与现有架构数据管理界面的对比

### 5.1 现有实现 vs 推荐 Pattern

| 区域 | 现有实现 | 推荐 Pattern | 改进点 |
|------|---------|-------------|--------|
| **Header** | AppHeader | WorkspaceHeader | 添加面包屑 |
| **Sidebar** | UnifiedScopePanel | WorkspaceSidebar + 独立组件 | 拆分职责 |
| **Tree** | 混合在 UnifiedScopePanel | ObjectTree + RelationTree | 独立组件，可复用 |
| **Main** | DynamicView | MainTabs + FilterBar + DynamicContent | 标准化 |
| **Detail** | 在 DynamicView 内部 | DetailPanel (侧滑) | 更流畅的交互 |
| **全局 Action** | Header 内 | GlobalActions 组件 | 更灵活的布局 |

### 5.2 需要新建的组件

1. **AppWorkspace.vue** - 工作空间顶层容器
2. **WorkspaceHeader.vue** - 全局 Header
3. **WorkspaceSidebar.vue** - 可调整宽度的侧边栏
4. **DimensionSelector.vue** - 维度选择器
5. **ObjectTree.vue** - 对象树（重构自 UnifiedScopePanel）
6. **RelationTree.vue** - 关系树
7. **MainTabs.vue** - 主 Tab 导航
8. **FilterBar.vue** - 过滤器栏（已有，需优化）
9. **DetailPanel.vue** - 详情面板（侧滑）

### 5.3 需要重构的组件

1. **UnifiedScopePanel.vue** → 拆分为 DimensionSelector + ObjectTree + RelationTree
2. **DynamicView.vue** → MainTabs + ListView + DetailPanel
