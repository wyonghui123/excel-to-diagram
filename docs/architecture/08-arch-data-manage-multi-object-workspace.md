## 一、核心交互流程

### 1.1 严格的三步选择流程

```
Step 1                    Step 2                      Step 3
┌─────────────────┐      ┌─────────────────┐       ┌─────────────────┐
│ 选择产品 + 版本   │ ───► │ 选择中心范畴      │ ───► │ 选择关系范围      │
│                 │      │ (对象树)         │       │ (关系树)         │
└─────────────────┘      └─────────────────┘       └─────────────────┘
        │                        │                        │
        ▼                        ▼                        ▼
┌─────────────────────────────────────────────────────────────────┐
│                        右侧主内容区                               │
│  展示符合当前过滤条件的所有对象数据 + 关系树                      │
│                                                                  │
│  ┌─────────┬─────────┬─────────┬─────────┬─────────┐        │
│  │ 领域     │ 子领域   │ 服务模块 │ 业务对象 │ 关系    │        │
│  └─────────┴─────────┴─────────┴─────────┴─────────┘        │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 每一步的详细说明

#### **Step 1: 选择产品 + 版本**

| 操作 | 结果 |
|------|------|
| 用户选择产品 | 显示该产品下的所有版本 |
| 用户选择版本 | 触发数据加载 |
| 加载完成后 | 显示：<br>- 完整的对象树（所有领域→子领域→服务模块→业务对象）<br>- 完整的关系树（所有关系类型）<br>- 主内容区显示所有对象数据 |

#### **Step 2: 在对象树中选择中心范畴**

| 操作 | 结果 |
|------|------|
| 用户在对象树中选择节点 | 确定"中心对象" |
| 系统基于中心对象过滤 | 过滤：<br>- 只显示中心对象及其下级<br>- 关系树高亮相关关系 |
| 主内容区更新 | 只显示中心对象及其下级对象的数据 |

**示例**：
- 如果选择"子领域 A"
- 显示：子领域 A + 其下的所有服务模块 + 业务对象
- 关系树：只显示与这些对象相关的关​​系

#### **Step 3: 在关系树中选择关系范围**

| 操作 | 结果 |
|------|------|
| 用户在关系树中选择关系类型 | 确定"关系范围" |
| 系统进一步过滤 | 只显示选中的关系类型 |
| 主内容区更新 | 只显示符合条件的关系 |

| 交互 | 行为 |
|------|------|
| 单击节点 | 选中该节点作为"中心范畴"，过滤数据 |
| 双击节点 | 展开/折叠子节点 |
| 右键节点 | 显示上下文菜单（查看详情、查看关系） |

## 五、数据流设计

### 5.1 状态管理

```javascript
// WorkspaceStore
{
  // Step 1: 产品和版本
  selectedProductId: null,
  selectedVersionId: null,
  
  // Step 2: 中心范畴（对象树选择）
  centerScope: {
    type: 'domain' | 'sub_domain' | 'service_module' | 'business_object',
    nodeId: null,       // 选中的节点 ID
    nodeIds: [],        // 多选时的节点 ID 列表
  },
  
  // Step 3: 关系范围（关系树选择）
  relationScope: {
    relationType: null,  // 选中的关系类型
    relationTypes: [],     // 多选时的关系类型列表
  },
  
  // 数据
  domains: [],
  subDomains: [],
  serviceModules: [],
  businessObjects: [],
  relationships: [],
  
  // 关系树
  relationTree: {},
  
  // 加载状态
  loading: false,
}
```

### 5.2 数据过滤逻辑

```javascript
// 计算最终的过滤条件
const finalFilter = computed(() => {
  // 基础条件：版本
  const base = { version_id: selectedVersionId.value }
  
  // Step 2: 中心范畴过滤
  if (centerScope.value.nodeId) {
    Object.assign(base, buildScopeFilter(centerScope.value))
  }
  
  // Step 3: 关系范围过滤
  if (relationScope.value.relationType) {
    Object.assign(base, { relation_type: relationScope.value.relationType })
  }
  
  return base
})

// 构建中心范畴过滤条件
function buildScopeFilter(scope) {
  switch (scope.type) {
    case 'domain':
      return { domain_id: scope.nodeId }
    case 'sub_domain':
      return { sub_domain_id: scope.nodeId }
    case 'service_module':
      return { service_module_id: scope.nodeId }
    case 'business_object':
      return { business_object_id: scope.nodeId }
    default:
      return {}
  }
}
```

### 5.3 关系树过滤逻辑

```javascript
// 根据中心范畴过滤关系树
const filteredRelationTree = computed(() => {
  if (!centerScope.value.nodeId) {
    return relationTree.value  // 返回完整关系树
  }
  
  // 获取中心对象及其下级对象
  const centerObjects = getDescendants(centerScope.value)
  
  // 过滤关系树，只保留与这些对象相关的关​​系
  return filterRelationTree(relationTree.value, centerObjects)
})
```
