## 二、问题修复详情

### 2.1 Bug 1: 关系统计回退问题

**问题**：未选择关系时，类型和配置步骤的关系统计显示数值（应显示 0）

**根因**：`filteredRelations.value.length || totalStats.objectRelations` 回退到总数

**修复文件**：`src/views/AADiagramApp/composables/useDiagramData.js`

**修复内容**：
```javascript
// incrementalStats.objectRelations
objectRelations: filteredRelations.value.length  // 移除 || totalStats.objectRelations

// displayStats.total.objectRelations
objectRelations: filteredRelations.value.length || 0

// configStats.objectRelations
objectRelations: filteredRelations.value.length || 0
```

---

### 2.2 Bug 2: 配置步骤默认选择规则

**问题**：没有关系选择时，区分中心范围应默认不选中

**修复文件**：`src/views/AADiagramApp/index.vue`

**修复内容**：
```javascript
watch(currentStep, (newStep, oldStep) => {
  if (newStep === 4) {
    const hasSelectedRelations = selectedRelationNodeIds.value && selectedRelationNodeIds.value.length > 0
    if (!hasSelectedRelations) {
      configStore.updateCenterScopeHighlight(false)
    }
  }
})
```

---

### 2.3 Bug 3: Legend 显示编码而非名称

**问题**：业务对象图在服务模块分组下 Legend 显示编码（如 "CG"）而非名称（如 "采购供应"）

**根因**：`groupKey = node.serviceModule || node.serviceModuleName` 优先取编码

**修复文件**：`src/composables/useMermaid/renderer/useSvgProcessor.js`

**修复内容**：
```javascript
// 修改前
groupKey = node.serviceModule || node.serviceModuleName

// 修改后
groupKey = node.serviceModuleName || node.serviceModule || node.name
```

---

### 2.4 Bug 4: 服务模块图 Legend 不显示

**问题**：服务模块图没有任何 Legend 显示

**根因**：三个问题叠加
1. SM Syntax 不返回 `nodeColorMappings`
2. MermaidComponent 不提取 SM Syntax 的 `nodeColorMappings`
3. `buildColorLegendData` 中 `colorGroupBy='serviceModule'` 分支缺少对 SM 节点结构的处理

**修复文件**：
1. `src/composables/useMermaid/syntax/useServiceModuleSyntax.js` - 添加 `nodeColorMappings` 返回
2. `src/components/MermaidComponent.vue` - 提取 `nodeColorMappings`
3. `src/composables/useMermaid/renderer/useSvgProcessor.js` - 添加 SM 节点分组支持

### 3.2 核心差异对比表

| 维度 | 业务对象图 (BO) | 服务模块图 (SM) | 影响 |
|------|:---:|:---:|------|
| **节点 ID** | `bo.name` | `sm.code` | Legend 匹配逻辑不同 |
| **节点 color** | ❌ 无（语法层计算） | ✅ 有（构建层计算） | Legend 取色路径不同 |
| **serviceModule 字段** | ✅ 有（编码） | ❌ 无 | groupKey 计算需回退 |
| **serviceModuleName 字段** | ✅ 有（名称） | ❌ 无 | groupKey 计算需回退 |
| **isCenter 计算** | 上游 `bo.isCenter` | `centerScopeHighlight && codes.has(sm.code)` | 判断时机不同 |
| **颜色计算时机** | 语法层（useBusinessObjectSyntax） | 构建层（serviceModuleDiagramBuilder） | 修改颜色需改不同位置 |
| **centerScope 传递** | 作为数组传递 | 不传递（用 centerServiceModuleCodes） | 中心范围判断方式不同 |

## 五、历史文档补充发现

### 5.1 containers vs children 的双重含义

| 职责 | containers | children |
|------|-----------|----------|
| **控制面板显示** | ✅ 用于渲染可拖拽节点列表 | ❌ 不用于显示 |
| **控制面板分组** | ❌ 不用于分组 | ✅ 用于递归渲染分组树 |
| **合并用户配置** | ❌ 不被合并 | ✅ 被合并 |
| **图表渲染（统一渲染器）** | ❌ 不使用 | ✅ 从 children 取终端节点 |
| **图表渲染（旧渲染器）** | ✅ 从 containers 取终端节点 | ❌ 不使用 |

### 5.2 两套渲染路径并行

| 路径 | 说明 |
|------|------|
| **统一渲染路径** | UnifiedRenderer + ColorCalculator + enrichGroupModel |
| **旧渲染路径** | diagramDataBuilder / serviceModuleDiagramBuilder + useBusinessObjectSyntax / useServiceModuleSyntax |
