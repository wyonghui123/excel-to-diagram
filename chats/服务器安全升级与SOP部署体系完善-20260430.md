## 2. 安全升级执行记录

### 2.1 Python 升级（3.9.17 → 3.9.21）

**遇到的问题**:
- 下载源无法访问（404 Not Found）
- 缺少依赖包（sqlite-devel、libffi-devel）
- Conda 环境冲突
- 编译时内存不足被系统 kill
- 缺少 _ctypes 模块（libffi 未安装）

**解决方案**:
1. 手动下载源码包上传到 `/opt/`
2. 安装 sqlite-devel-3.7.17-8.el7.x86_64.rpm
3. 创建隔离启动脚本 `python3.9.21-clean`
4. 关闭 debug 模式避免 _ctypes 依赖

**关键命令**:
```bash
# 编译安装 Python 3.9.21
cd /opt/Python-3.9.21
./configure --prefix=/opt/python3.9.21
make
make install

# 创建隔离脚本
cat > /usr/local/bin/python3.9.21-clean << 'EOF'
#!/bin/bash
env -u PYTHONPATH -u PYTHONHOME -u CONDA_DEFAULT_ENV \
    -u CONDA_PREFIX -u CONDA_PROMPT_MODIFIER \
    -u CONDA_EXE -u CONDA_PYTHON_EXE \
    /opt/python3.9.21/bin/python3.9 "$@"
EOF
chmod +x /usr/local/bin/python3.9.21-clean
```

### 2.2 OpenSSH 升级（7.4p1 → 9.9p1）

**遇到的问题**:
- PAM headers 缺失
- SSH 密钥权限问题
- 堡垒机连接中断
- 不支持的配置选项（GSSAPIAuthentication）

**解决方案**:
1. 安装 pam-devel-1.1.8-22.el7.x86_64.rpm
2. 修复密钥权限 `chmod 600 /etc/ssh/ssh_host_*_key`
3. 修改 PAM 配置允许 root 登录
4. 注释不支持的配置选项

### 2.3 OpenSSL 升级（旧版本 → 1.1.1u）

```bash
# 安装到独立目录避免冲突
./config --prefix=/usr/local/openssl shared zlib
make && make install
```

### 2.4 zlib 升级（旧版本 → 1.3.1）

```bash
./configure --prefix=/usr/local
make && make install
```

## 3. 应用服务配置

### 3.1 服务架构

```
前端服务 (8081)
  ├── 工作目录: /opt/app
  ├── 启动脚本: server.py
  └── 日志: /opt/app/shared/logs/frontend.log

后端服务 (5000)
  ├── 工作目录: /opt/app/meta
  ├── 启动脚本: server.py
  └── 日志: /opt/app/shared/logs/backend.log
```

### 3.2 端口配置变更

**问题**: 前端配置指向 5001，后端实际运行在 5000

**解决**:
```bash
# 修改前端代理配置
sed -i 's/localhost:5001/localhost:5000/g' /opt/app/server.py
```

### 3.3 启动命令

```bash
# 后端
cd /opt/app/meta
nohup /usr/local/bin/python3.9.21-clean server.py > /opt/app/shared/logs/backend.log 2>&1 &

# 前端
cd /opt/app
nohup /usr/local/bin/python3.9.21-clean server.py > /opt/app/shared/logs/frontend.log 2>&1 &
```

## 5. Docker 化评估

### 5.1 评估结论

**暂缓 Docker 化，理由**:
1. 当前 SOP 方案已满足需求
2. Docker 化增量价值有限
3. 投入产出比不高
4. 内网环境增加复杂度

### 5.2 价值对比

| 方面 | 现状（SOP） | Docker 化 | 增量价值 |
|------|-------------|-----------|----------|
| 环境一致性 | ❌ | ✅ | 中等 |
| 部署速度 | 5-10分钟 | 3-5分钟 | 有限 |
| 学习成本 | 低 | 高 | 负向 |
| 维护成本 | 低 | 高 | 负向 |

### 5.3 触发条件（满足任一则重新评估）

- [ ] 部署成功率 < 95%
- [ ] 需要管理多台服务器
- [ ] 需要支持多个开发者
- [ ] 环境不一致问题频繁出现
