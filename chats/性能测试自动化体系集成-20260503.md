## 1. 问题修复：user_group_api.py 变量名错误

### 问题描述
运行性能测试时出现 `NameError: name 'bp' is not defined` 错误。

### 根本原因
[user_group_api.py](file:///d:/filework/excel-to-diagram/meta/api/user_group_api.py) 文件中定义了 `user_group_bp` 变量，但在路由装饰器中错误地使用了 `@bp.route`（10处）。

### 修复内容
将所有 `@bp.route` 替换为 `@user_group_bp.route`：

```python
# 修复前
@bp.route('/user-groups', methods=['POST'])

# 修复后
@user_group_bp.route('/user-groups', methods=['POST'])
```

### 测试验证
修复后运行全部性能测试：
```
pytest meta/tests/performance/ -v --tb=short -q
36 passed in 102.25s (0:01:42)
```

## 4. CI/CD 工作流配置

### .github/workflows/tests.yml

```yaml
on:
  push:
    branches: [ main, master, develop ]
  pull_request:
    branches: [ main, master, develop ]
  schedule:
    - cron: '0 2 * * *'  # 每天凌晨2点

jobs:
  unit-tests:        # 推送到主分支时运行
  integration-tests: # PR 时运行
  api-tests:         # PR 时运行
  performance-tests:  # PR 或定时触发
  full-test-suite:   # 合并到 main 时运行
```

## 5. 本地开发命令

```bash
# ========== 日常开发 (快速反馈) ==========
make test-unit          # 仅单元测试 (~5s)
pytest -m unit -v

# ========== 提交前验证 (中等覆盖) ==========
make test               # 常规测试 (~30s)
pytest -m "unit or integration" -v

# ========== API 开发验证 ==========
make test-api           # API 测试 (~15s)
pytest -m api -v

# ========== 性能验证 (发布前) ==========
make test-performance   # 性能测试 (~2min)
pytest meta/tests/performance/ -v

# ========== 完整验证 (发布前) ==========
make test-all           # 全量测试 (~3min)
pytest --cov=meta -v

# ========== 覆盖率报告 ==========
make test-coverage      # 生成 HTML 报告
```

## 8. 自动化执行机制说明

### 当前自动化程度

| 层级 | 机制 | 说明 |
|------|------|------|
| Level 3 | CI/CD 自动触发 | Push、PR、定时任务自动运行 |
| Level 2 | Git Hooks | 可配置，需手动启用 |
| Level 1 | 手动触发 | `make test-*` 命令 |

### CI/CD 自动触发流程

```
git push origin <branch>
         │
         ▼
  ┌─────────────────┐
  │ GitHub Actions  │ ◄── 自动触发
  └────────┬────────┘
           │
           ├──► unit-tests        (自动运行)
           ├──► integration-tests  (自动运行)
           ├──► api-tests          (自动运行)
           └──► performance-tests  (PR/定时触发)
```

### 无需手动操作的场景

| 场景 | 触发条件 | 结果通知 |
|------|----------|----------|
| 代码推送 | `git push` | GitHub Actions 状态 |
| PR 创建/更新 | Push 新提交 | PR 检查状态 |
| 每日构建 | 凌晨 2:00 | GitHub Artifacts |

### 监控位置

```
GitHub 仓库 → Actions 标签页
├── Workflows
│   ├── Tests
│   └── performance-tests (报告)
└── Artifacts (性能报告下载)
```
