# ELK布局引擎嵌套容器间距问题排查

**日期**: 2025-01-20

## 问题描述

在使用Mermaid的ELK布局引擎渲染嵌套subgraph（容器）时，嵌套容器的左右两边空间太少，节点和容器边界之间没有足够的间距。

## 问题背景

- 项目使用Vue.js + Mermaid.js进行图表渲染
- 支持Dagre和ELK两种布局引擎
- ELK布局引擎在处理嵌套subgraph时，内部间距控制不如预期

## 排查过程

### 1. 初始配置检查

查看 `useMermaidConfig.js` 中的ELK配置：

```javascript
const elkOptions = {
  'elk.spacing.nodeNode': 80,
  'elk.layered.spacing.nodeNodeBetweenLayers': 120,
  'elk.padding': '[top:25, left:30, right:30, bottom:25]',
  'elk.hierarchyHandling': 'INCLUDE_CHILDREN',
  'elk.algorithm': 'layered',
  // ...
}
```

### 2. 发现问题：padding格式错误

通过查阅ELK官方文档，发现padding格式应该使用 `=` 分隔符，而不是 `:`：

```javascript
// 错误格式
'elk.padding': '[top:25, left:30, right:30, bottom:25]'

// 正确格式
'elk.padding': '[top=40,left=50,right=50,bottom=40]'
```

### 3. 尝试多种配置方式

#### 方式1：通过 `mermaid.initialize()` 配置

修改 `getElkConfig()` 函数：

```javascript
const elkOptions = {
  'elk.spacing.nodeNode': 100,
  'elk.layered.spacing.nodeNodeBetweenLayers': 150,
  'elk.padding': '[top=40,left=50,right=50,bottom=40]',
  'elk.hierarchyHandling': 'INCLUDE_CHILDREN',
  'elk.algorithm': 'layered',
  'elk.layered.crossingMinimization.strategy': 'LAYER_SWEEP',
  'elk.layered.spacing.edgeNodeBetweenLayers': 60,
  'elk.nodeSpacing.selfLoop': 50,
  'elk.layered.spacing.componentComponent': 80,
  'elk.spacing.componentComponent': 80,
  'elk.aspectRatio': '1.6',
  'elk.contentAlignment': 'H_CENTER',
}
```

**结果**: 没有效果

#### 方式2：通过 `%%{init: ...}%%` 指令

在Mermaid代码开头添加init指令：

```javascript
elkInitDirective = `%%{init: {"flowchart": {"elk": {"spacing.nodeNode": 100, ...}}}}%%\n`
```

**结果**: 没有效果

#### 方式3：通过YAML frontmatter配置

```yaml
---
config:
  flowchart:
    nodeSpacing: 100
    rankSpacing: 150
    diagramPadding: 50
    padding: 30
---
```

**结果**: 没有效果

### 4. 研究Mermaid ELK配置机制

通过查阅Mermaid源码和文档，发现：

1. Mermaid的ELK配置通过 `@mermaid-js/layout-elk` 包实现
2. Mermaid暴露的ELK配置选项有限，主要包括：
   - `nodePlacementStrategy`
   - `cycleBreakingStrategy`
   - `mergeEdges`
   - `forceNodeModelOrder`
   - `considerModelOrder`

3. ELK原生的spacing/padding配置可能没有完全暴露给Mermaid用户

### 5. 查看SVG后处理逻辑

发现 `useSvgProcessor.js` 中有 `fixElkContainerBounds` 函数，用于在渲染后调整容器边界：

```javascript
const fixElkContainerBounds = (svgEl, layoutEngine) => {
  if (layoutEngine !== 'elk') return
  
  const minGap = 35
  const padding = 25
  
  // 调整嵌套容器间距和边界
  // ...
}
```

## 关键发现

1. **Mermaid的ELK集成限制**: Mermaid没有完全暴露ELK的所有配置选项，特别是spacing和padding相关的参数

2. **配置传递问题**: 即使在 `mermaid.initialize()` 中设置了ELK参数，这些参数可能没有被正确传递给ELK渲染器

3. **SVG后处理**: 项目中已有SVG后处理逻辑来调整容器边界，但可能需要进一步增强

## 可能的解决方案

### 方案A：增强SVG后处理

在 `fixElkContainerBounds` 函数中增加更激进的间距调整逻辑：

```javascript
// 增加容器内部左右padding
const horizontalPadding = 40;
childRect.setAttribute('x', childX + horizontalPadding);
parentRect.setAttribute('width', parentWidth + horizontalPadding * 2);
```

### 方案B：修改ELK渲染器

直接修改 `@mermaid-js/layout-elk` 包，在构建ELK图时注入spacing配置。

### 方案C：使用Dagre引擎

如果ELK的间距控制无法满足需求，可以考虑使用Dagre引擎作为替代。
