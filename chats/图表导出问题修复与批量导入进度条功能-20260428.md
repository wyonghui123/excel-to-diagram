## 一、图表HTML导出 "Maximum text size in diagram exceeded" 问题

### 问题描述
用户下载HTML图表后打开，出现 "Maximum text size in diagram exceeded" 错误，且提示需要从CDN加载资源。

### 根因分析
- Mermaid 的 `maxTextSize` 参数限制图表定义的总文本大小为 **90,000 字符**
- 图表节点多、标签长时，Mermaid 代码总长度超限导致渲染失败
- 错误发生在 [MermaidComponent.vue:1133](file:///d:/filework/excel-to-diagram/src/components/MermaidComponent.vue#L1133) 的 HTML 导出模板配置中

### 修复方案
将 `maxTextSize` 从 `90,000` 提升到 `500,000`，修改了 3 处：

| 文件 | 行号 | 用途 |
|------|------|------|
| `src/composables/useMermaid/config/useMermaidConfig.js` | L144 | 共享配置（内联渲染使用） |
| `src/components/MermaidComponent.vue` | L808 | 内联渲染器配置 |
| `src/components/MermaidComponent.vue` | L1133 | **HTML导出模板配置（本次问题的关键位置）** |

同时在 HTML 导出模板中为 `mermaid.run()` 添加了 `.catch()` 错误处理，当文本超限时显示友好提示。

## 三、批量导入进度条功能

### 需求
架构数据管理批量导入时，增加进度条显示。

### 原有流程问题
- 导入过程是**同步阻塞**的 HTTP POST 请求
- 前端只有 spinner 显示"正在导入..."，用户不知道进展
- 大文件导入时用户可能以为卡死了

### 方案设计：轮询模式

#### 数据流
```
前端点"开始导入"
  → POST /import/async (立即返回 task_id)
  → 后台线程执行 import_cascade(progress_callback=...)
      → 每处理完一个类型 → 回调更新 _import_tasks[task_id]
  → 前端每1秒 GET /import/status/{task_id} 轮询
  → 更新进度条 UI
  → status=completed → 显示结果表格
```

#### 进度节点
`import_cascade()` 按 `import_order` 逐类型处理（domain → sub_domain → service_module → business_object），每个类型完成就是一个天然进度节点（通常 4~5 个）。

### 实现改动

#### 后端 API（`meta/api/export_import_api.py`）
- 新增 `_import_tasks` 内存字典存储任务状态
- 新增 `POST /api/v1/import/async` — 异步导入接口，立即返回 task_id，后台线程执行
- 新增 `GET /api/v1/import/status/<task_id>` — 轮询接口，返回 {status, progress, current_type_name, message, result}
- 新增 `_update_task_progress()` 辅助函数

#### 后端服务（`meta/services/import_export_service.py`）
- `import_cascade()` 新增 `progress_callback` 参数
- 在 execute 模式的循环中：开始前、每个类型处理**后**、全部完成后各调用一次回调
- 回调数据包含：progress(0-100), current_type, current_type_name, total_types, current_index, message

#### 前端 API（`src/views/ArchDataManageApp/composables/useApi.js`）
- 新增 `importDataAsync()` — 调用 /import/async 接口
- 新增 `getImportStatus(taskId)` — 调用 /import/status 接口

#### 前端组件（`src/views/ArchDataManageApp/components/ImportDialog.vue`）
- Step3 从 spinner 替换为**进度条 UI**（进度百分比 + 条形图 + 当前类型详情）
- `handleImport()` 改为异步模式：提交任务 → 1秒轮询 → 更新进度 → 完成/失败展示
- 新增状态变量：importStatus, progress, progressMessage, currentTypeName 等
- 进度中禁用关闭按钮，完成/失败时显示"关闭"
- `handleCancel` 清理轮询定时器

### 图2：进度条在最后一步直接100%

**问题**：处理关系(5/5)时，进度直接跳到100%，但实际还在跑。

**根因**：进度回调在类型**开始前**就推进了进度。

**修复**：改为**处理后**才推进进度

```python
# 之前（错误）：开始时就推进
current_progress = int((current_index / total_types) * 100)
progress_callback({'progress': current_progress, ...})
# 处理中...
sheet_result = self._import_sheet(...)
progress_callback({'progress': current_progress, ...})  # 完成后没有再更新

# 之后（正确）：完成后才推进
progress_callback({'progress': old_progress, 'message': '正在导入: xxx'})  # 开始时保持上一轮进度
sheet_result = self._import_sheet(...)
completed_count += 1
progress_callback({'progress': int((completed_count / total_types) * 100), 'message': '已完成: xxx'})  # 完成后推进
```

**改动**：[import_export_service.py:2088-2146](file:///d:/work/excel-to-diagram/meta/services/import_export_service.py#L2088-L2146)
