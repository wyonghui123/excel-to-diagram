3 · 入门：5 步挂上第一个对象

  1. 进入「架构数据管理」，左侧为层级树。
  2. 点击目标「子领域」节点，右侧出列表。
  3. 列表右上「+ 新建业务对象」。
  4. 填写编码、名称、责任人，必填项标红星。
  5. 保存后对象自动归属到该子领域，路径：产品线 / 版本 / 领域 / 子领域 / 业务对象。
提示：对象编码建议与代码仓库中的实体名保持一致，便于溯源。

4 · 探索：常用操作

操作快捷入口适用场景
模糊搜索顶栏 🔍只知道名称/编码关键字
高级筛选列表上方「筛选」按责任人/状态/标签多条件
批量编辑勾选 → 「批量操作」同一负责人的对象统一改派
拖拽移动树节点按住拖动调整父子归属
导出 Excel列表右上「⤓」线下评审或备份

5 · 进阶：版本快照与对比

每次产品版本发布时，系统会自动生成一份快照。快照不可修改，只能对比。

  • 路径：对象详情 → 「历史版本」。
  • 对比维度：字段值变更、归属路径变更、负责人变更。
  • 对比结果可导出 PDF，作为变更评审的附件。
权限提示：查看历史快照需要 arch:history:view，导出 PDF 需要 arch:export。

6 · 实战：V8.0 → V8.1 迁移

需求：V8.0 的"供应链"子领域拆分为"采购"和"仓储"两个子领域，业务对象按 7:3 比例迁移。

步骤操作结果
1在 V8.1 下新建「采购」「仓储」两个子领域空节点
2用「批量移动」勾选 7 条对象 → 采购7 条归属变更
3同样操作 3 条 → 仓储合计 10 条迁移完成
4V8.0 的"供应链"节点标记为「归档」默认不显示
5点击「生成变更报告」PDF 自动下载
