3 · 入门：5 步建第一个角色

  1. 进入「系统管理 → 角色管理」。
  2. 右上「+ 新建角色」，输入名称（如 产品经理）与编码。
  3. 切到「菜单权限」标签，勾选该角色可见的菜单。
  4. 切到「数据范围」标签，按需设置"全部/本部门/本用户"。
  5. 保存，把用户拖到该角色的成员列表。
原则：角色用「能做什么」描述（如 product_manager），不要用部门名。

4 · 探索：菜单权限 vs 数据权限

维度菜单权限数据权限
管什么页面/按钮可见性同页面内能看哪些行
颗粒度菜单项 + 操作按钮行级 / 列级 / 字段级
典型场景"不能进系统配置页""只能看本部门订单"
存储位置role.menu_perms (JSON)role.data_scope (DSL)

5 · 进阶：字段级权限

字段级权限用于"同一行数据，不同角色看到不同字段"——例如员工表里薪资字段仅 HR 可见。

  • 配置位置：业务对象详情 → 「字段权限」标签。
  • 支持 READ / WRITE / HIDDEN 三种策略。
  • 优先级：HIDDEN > WRITE > READ；多角色取并集。
小心：HIDDEN 不是加密，后端 API 仍会返回；务必配合服务端校验。

6 · 实战：财务仅看本部门

需求：财务角色 finance_user 登录后只能看本部门的报销单。

步骤操作DSL / 配置
1新建角色 finance_user勾选「报销管理」菜单
2数据范围选「本部门」scope = DEPT
3报销对象字段权限金额 → READ；备注 → HIDDEN
4把 3 个财务账号加入用户列表拖入
5财务登录验证只能看到本部门 + 金额不能改
