3 · 入门：3 步查第一条记录

  1. 左侧菜单「审计日志」。
  2. 顶部筛选器：选时间范围 + 业务对象。
  3. 列表点击行展开，可看到「旧值 → 新值」diff。
提示：审计日志只读不可改；删除需走合规审批流程。

4 · 探索：日志字段速查

字段含义典型值
trace_id同一请求链路 ID7f3a...d92
actor操作者user_id / system
action操作类型CREATE / UPDATE / DELETE / READ
object_type对象类型product / order / user
object_id对象主键123
diff字段值变更{price: [100, 120]}
ts时间戳（毫秒）1718892000123

5 · 进阶：合规报表与告警

  • 批量导出：选时间范围导出 CSV / Parquet，最大支持 100 万行/次。
  • 合规报表：预置 SOX / 等保模板，10 分钟生成月报。
  • 告警钩子：DELETE 操作可配置触发邮件 / 飞书 / 钉钉通知。
合规提示：审计数据保存期限默认 365 天，到期自动归档冷存储（可配置 730 / 1095 天）。

6 · 实战：定位"误删采购订单"

步骤操作结果
1筛选 action=DELETE + object_type=order命中 3 条
2按时间倒序，点最新一条trace_id = 7f3a...d92
3用 trace_id 全局搜索同链路还有 1 条 UPDATE
4展开 UPDATE diffstatus 从「待审」→「已删」
5查看 actor 详情操作人 + 登录 IP + UA

结论：5 分钟内定位到具体人、具体时间、具体字段，可作为合规证据提交。
